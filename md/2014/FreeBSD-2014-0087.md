# qt4-imageformats, qt5-gui -- DoS vulnerability in the GIF image handler
Richard J. Moore reports:

> The builtin GIF decoder in QtGui prior to Qt 5.3 contained a bug that
> would lead to a null pointer dereference when loading certain hand
> crafted corrupt GIF files. This in turn would cause the application
> loading these hand crafted GIFs to crash.


## Affected packages
 - **qt4-imageformats**
    - **Fixed:** 4.8.6_1
 - **qt5-gui**
    - **Fixed:** 5.2.1_4
