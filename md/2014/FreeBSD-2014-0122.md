# bash -- remote code execution
Note that this is different than the public \"Shellshock\" issue.

Specially crafted environment variables could lead to remote arbitrary
code execution. This was fixed in bash 4.3.27, however the port was
patched with a mitigation in 4.3.25_2.


## Affected packages
 - **bash**
    - **Fixed:** 4.3.25_2
 - **bash-static**
    - **Fixed:** 4.3.25_2
