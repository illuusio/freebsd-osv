# gnupg -- possible DoS using garbled compressed data packets
Werner Koch reports:

> This release includes a \*security fix\* to stop a possible DoS using
> garbled compressed data packets which can be used to put gpg into an
> infinite loop.


## Affected packages
 - **gnupg1**
    - **Fixed:** 1.4.17
 - **gnupg**
    - **Fixed:** 2.0.24
