# asterisk -- Long Contact URIs in REGISTER requests can crash Asterisk
The Asterisk project reports:

> Asterisk may crash when processing an incoming REGISTER request if
> that REGISTER contains a Contact header with a lengthy URI.
>
> This crash will only happen for requests that pass authentication.
> Unauthenticated REGISTER requests will not result in a crash
> occurring.
>
> This vulnerability only affects Asterisk when using PJSIP as its SIP
> stack. The chan_sip module does not have this problem.


## Affected packages
 - **asterisk13**
    - **Fixed:** 13.8.1
