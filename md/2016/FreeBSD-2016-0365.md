# inspircd -- authentication bypass vulnerability
Adam reports:

> A serious vulnerability exists in when using m_sasl in combination
> with any services that support SASL EXTERNAL. To be vulnerable you
> must have m_sasl loaded, and have services which support SASL EXTERNAL
> authentication.


## Affected packages
 - **inspircd**
    - **Fixed:** 2.0.23
