# py-cryptography -- vulnerable HKDF key generation
Alex Gaynor reports:

> Fixed a bug where \`\`HKDF\`\` would return an empty byte-string if
> used with a \`\`length\`\` less than \`\`algorithm.digest_size\`\`.


## Affected packages
 - **py27-cryptography**
    - **Fixed:** 1.5.3
 - **py33-cryptography**
    - **Fixed:** 1.5.3
 - **py34-cryptography**
    - **Fixed:** 1.5.3
 - **py35-cryptography**
    - **Fixed:** 1.5.3
