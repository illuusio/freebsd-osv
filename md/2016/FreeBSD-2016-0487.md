# vim -- arbitrary command execution
Mitre reports:

> vim before patch 8.0.0056 does not properly validate values for the
> \'filetype\', \'syntax\' and \'keymap\' options, which may result in
> the execution of arbitrary code if a file with a specially crafted
> modeline is opened.


## Affected packages
 - **vim**
    - **Fixed:** 8.0.0056
 - **vim-console**
    - **Fixed:** 8.0.0056
 - **vim-lite**
    - **Fixed:** 8.0.0056
 - **neovim**
    - **Fixed:** 0.1.7
