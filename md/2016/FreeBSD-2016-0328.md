# FreeBSD -- Integer overflow in IGMP protocol
# Problem Description:

An integer overflow in computing the size of IGMPv3 data buffer can
result in a buffer which is too small for the requested operation.

# Impact:

An attacker who can send specifically crafted IGMP packets could cause a
denial of service situation by causing the kernel to crash.


## Affected packages
 - **FreeBSD-kernel**
    - **Introduced:** 10.1
    - **Fixed:** 10.1_9
    - **Introduced:** 9.3
    - **Fixed:** 9.3_13
    - **Introduced:** 8.4
