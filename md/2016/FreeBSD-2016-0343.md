# FreeBSD -- Insecure default snmpd.config permissions
# Problem Description:

The SNMP protocol supports an authentication model called USM, which
relies on a shared secret. The default permission of the snmpd
configuration file, /etc/snmpd.config, is weak and does not provide
adequate protection against local unprivileged users.

# Impact:

A local user may be able to read the shared secret, if configured and
used by the system administrator.
