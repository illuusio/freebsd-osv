# qemu -- denial of service vulnerability in IDE disk/CD/DVD-ROM emulation
Prasad J Pandit, Red Hat Product Security Team, reports:

> Qemu emulator built with the IDE disk and CD/DVD-ROM emulation support
> is vulnerable to a divide by zero issue. It could occur while
> executing an IDE command WIN_READ_NATIVE_MAX to determine the maximum
> size of a drive.
>
> A privileged user inside guest could use this flaw to crash the Qemu
> instance resulting in DoS.


## Affected packages
 - **qemu**
    - **Fixed:** 2.4.1
 - **qemu-devel**
    - **Fixed:** 2.4.1
 - **qemu-sbruno**
    - **Fixed:** 2.5.50.g20151224
 - **qemu-user-static**
    - **Fixed:** 2.5.50.g20151224
