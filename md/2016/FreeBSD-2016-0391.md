# redis -- sensitive information leak through command history file
Redis team reports:

> The redis-cli history file (in linenoise) is created with the default
> OS umask value which makes it world readable in most systems and could
> potentially expose authentication credentials to other users.


## Affected packages
 - **redis**
    - **Fixed:** 3.2.3
 - **redis-devel**
    - **Fixed:** 3.2.3
