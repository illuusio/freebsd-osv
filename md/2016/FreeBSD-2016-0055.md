# privoxy -- malicious server spoofing as proxy vulnerability
Privoxy Developers reports:

> Proxy authentication headers are removed unless the new directive
> enable-proxy-authentication-forwarding is used. Forwarding the headers
> potentially allows malicious sites to trick the user into providing
> them with login information. Reported by Chris John Riley.


## Affected packages
 - **privoxy**
    - **Fixed:** 3.0.21
