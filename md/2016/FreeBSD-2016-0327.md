# FreeBSD -- SCTP stream reset vulnerability
# Problem Description:

The input validation of received SCTP RE_CONFIG chunks is insufficient,
and can result in a NULL pointer deference later.

# Impact:

A remote attacker who can send a malformed SCTP packet to a FreeBSD
system that serves SCTP can cause a kernel panic, resulting in a Denial
of Service.
