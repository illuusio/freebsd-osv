# FreeBSD -- Linux compatibility layer issetugid(2) system call
# Problem Description:

A programming error in the Linux compatibility layer could cause the
issetugid(2) system call to return incorrect information.

# Impact:

If an application relies on output of the issetugid(2) system call and
that information is incorrect, this could lead to a privilege
escalation.


## Affected packages
 - **FreeBSD-kernel**
    - **Introduced:** 10.2
    - **Fixed:** 10.2_11
    - **Introduced:** 10.1
    - **Fixed:** 10.1_28
    - **Introduced:** 9.3
    - **Fixed:** 9.3_35
