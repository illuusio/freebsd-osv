# FreeBSD -- OpenSSL Remote DoS vulnerability
# Problem Description:

Due to improper handling of alert packets, OpenSSL would consume an
excessive amount of CPU time processing undefined alert messages.

# Impact:

A remote attacker who can initiate handshakes with an OpenSSL based
server can cause the server to consume a lot of computation power with
very little bandwidth usage, and may be able to use this technique in a
leveraged Denial of Service attack.
