# jpgraph2 -- XSS vulnerability
Martin Barbella reports:

> JpGraph is an object oriented library for PHP that can be used to
> create various types of graphs which also contains support for client
> side image maps. The GetURLArguments function for the JpGraph\'s Graph
> class does not properly sanitize the names of get and post variables,
> leading to a cross site scripting vulnerability.


## Affected packages
 - **jpgraph2**
    - **Fixed:** 3.0.7_1
