# librsync -- collision vulnerability
Michael Samuel reports:

> librsync before 1.0.0 uses a truncated MD4 checksum to match blocks,
> which makes it easier for remote attackers to modify transmitted data
> via a birthday attack.


## Affected packages
 - **librsync**
    - **Fixed:** 1.0.0
