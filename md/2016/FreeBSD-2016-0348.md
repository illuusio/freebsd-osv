# FreeBSD -- Incorrect argument handling in sendmsg(2)
# Problem Description:

Incorrect argument handling in the socket code allows malicious local
user to overwrite large portion of the kernel memory.

# Impact:

Malicious local user may crash kernel or execute arbitrary code in the
kernel, potentially gaining superuser privileges.


## Affected packages
 - **FreeBSD-kernel**
    - **Introduced:** 10.3
    - **Fixed:** 10.3_3
    - **Introduced:** 10.2
    - **Fixed:** 10.2_17
    - **Introduced:** 10.1
    - **Fixed:** 10.1_34
