# bind8 negative cache poison attack
A programming error in BIND 8 named can result in a DNS message being
incorrectly cached as a negative response. As a result, an attacker may
arrange for malicious DNS messages to be delivered to a target name
server, and cause that name server to cache a negative response for some
target domain name. The name server would thereafter respond negatively
to legitimate queries for that domain name, resulting in a
denial-of-service for applications that require DNS.
