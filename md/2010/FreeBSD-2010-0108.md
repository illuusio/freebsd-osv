# monotone -- remote denial of service in default setup
The monotone developers report:

> Running \"mtn \'\'\" or \"mtn ls \'\'\" doesn\'t cause an internal
> error anymore. In monotone 0.48 and earlier this behavior could be
> used to crash a server remotely (but only if it was configured to
> allow execution of remote commands).


## Affected packages
 - **monotone**
    - **Fixed:** 0.48.1
