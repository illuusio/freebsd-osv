# lighttpd -- denial of service vulnerability
Lighttpd security advisory reports:

> If you send the request data very slow (e.g. sleep 0.01 after each
> byte), lighttpd will easily use all available memory and die
> (especially for parallel requests), allowing a DoS within minutes.


## Affected packages
 - **lighttpd**
    - **Fixed:** 1.4.26
