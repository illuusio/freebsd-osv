# slim -- insecure PATH assignment
SLiM assigns logged on users a PATH in which the current working
directory (\"./\") is included. This PATH can allow unintentional code
execution through planted binaries and has therefore been fixed SLiM
version 1.3.2.


## Affected packages
 - **slim**
    - **Fixed:** 1.3.2
