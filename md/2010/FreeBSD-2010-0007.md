# squid -- Denial of Service vulnerability in DNS handling
Squid security advisory 2010:1 reports:

> Due to incorrect data validation Squid is vulnerable to a denial of
> service attack when processing specially crafted DNS packets.
>
> This problem allows any trusted client or external server who can
> determine the squid receiving port to perform a short-term denial of
> service attack on the Squid service.


## Affected packages
 - **squid**
    - **Introduced:** 2.7.1
