# irssi -- multiple vulnerabilities
Two vulnerabilities have found in irssi. The first issue could allow
man-in-the-middle attacks due to a missing comparison of SSL server
hostnames and the certificate domain names (e.g. CN).

A second vulnerability, related to the nick matching code, could be
triggered by remote attackers in order to crash an irssi client when
leaving a channel.


## Affected packages
 - **irssi**
    - **Fixed:** 0.8.15
 - **zh-irssi**
    - **Fixed:** 0.8.15
 - **irssi-devel**
    - **Fixed:** 20100325
