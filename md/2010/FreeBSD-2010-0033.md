# krb5 -- remote denial of service vulnerability
An authenticated remote attacker can causing a denial of service by
using a newer version of the kadmin protocol than the server supports.

The MIT Kerberos team also reports the cause:

> The Kerberos administration daemon (kadmind) can crash due to
> referencing freed memory.


## Affected packages
 - **krb5**
    - **Fixed:** 1.6.3_9
