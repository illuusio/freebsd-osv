# ejabberd -- queue overload denial of service vulnerability
The Red Hat security response team reports:

> A remotely exploitable DoS from XMPP client to ejabberd server via too
> many \"client2server\" messages (causing the message queue on the
> server to get overloaded, leading to server crash) has been found.


## Affected packages
 - **ejabberd**
    - **Fixed:** 2.1.3
