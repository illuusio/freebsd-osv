# otrs -- SQL injection
OTRS Security Advisory reports:

> Missing security quoting for SQL statements allows agents and
> customers to manipulate SQL queries. So it\'s possible for
> authenticated users to inject SQL queries via string manipulation of
> statements.
>
> A malicious user may be able to manipulate SQL queries to read or
> modify records in the database. This way it could also be possible to
> get access to more permissions (e. g. administrator permissions).
>
> To use this vulnerability the malicious user needs to have a valid
> Agent- or Customer-session.


## Affected packages
 - **otrs**
    - **Fixed:** 2.4.7
