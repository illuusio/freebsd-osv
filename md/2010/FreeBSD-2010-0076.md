# libmspack -- infinite loop denial of service
There is a denial of service vulnerability in libmspack. The libmspack
code is built into cabextract, so it is also vulnerable.

Secunia reports:

> The vulnerability is caused due to an error when copying data from an
> uncompressed block (block type 0) and can be exploited to trigger an
> infinite loop by tricking an application using the library into
> processing specially crafted MS-ZIP archives.


## Affected packages
 - **libmspack**
    - **Fixed:** 0.0.20060920
 - **cabextract**
    - **Fixed:** 1.3
