# mDNSResponder -- corrupted stack crash when parsing bad resolv.conf
Juli Mallett reports:

> mdnsd will crash on some systems with a corrupt stack and once that\'s
> fixed it will still leak a file descriptor when parsing resolv.conf.
> The crash is because scanf is used with %10s for a buffer that is only
> 10 chars long. The buffer size needs increased to 11 chars to hold the
> trailing NUL. To fix the leak, an fclose needs added.


## Affected packages
 - **mDNSResponder**
