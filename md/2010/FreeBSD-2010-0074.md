# git -- buffer overflow vulnerability
Greg Brockman reports:

> If an attacker were to create a crafted working copy where the user
> runs any git command, the attacker could force execution of arbitrary
> code.


## Affected packages
 - **git**
    - **Introduced:** 1.5.6
    - **Fixed:** 1.7.1.1_1
