# asterisk -- Crash when negotiating for T.38 with a declined stream
The Asterisk project reports:

> When Asterisk sends a re-invite initiating T.38 faxing, and the
> endpoint responds with a declined media stream a crash will then occur
> in Asterisk.


## Affected packages
 - **asterisk15**
    - **Fixed:** 15.7.4
 - **asterisk16**
    - **Fixed:** 16.5.1
