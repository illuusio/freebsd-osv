# FreeBSD -- System call kernel data register leak
# Problem Description:

The callee-save registers are used by kernel and for some of them (%r8,
%r10, and for non-PTI configurations, %r9) the content is not sanitized
before return from syscalls, potentially leaking sensitive information.

# Impact:

Typically an address of some kernel data structure used in the syscall
implementation, is exposed.


## Affected packages
 - **FreeBSD-kernel**
    - **Introduced:** 12.0
    - **Fixed:** 12.0_3
    - **Introduced:** 11.2
    - **Fixed:** 11.2_9
