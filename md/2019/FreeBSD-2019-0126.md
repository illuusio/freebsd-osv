# vlc -- Buffer overflow vulnerability
zhangyang reports:

> The ReadFrame function in the avi.c file uses a variable
> i_width_bytes, which is obtained directly from the file. It is a
> signed integer. It does not do a strict check before the memory
> operation(memmove, memcpy), which may cause a buffer overflow.


## Affected packages
 - **vlc**
    - **Fixed:** 3.0.7,4
