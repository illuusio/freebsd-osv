# sqlite3 -- use after free
MITRE reports:

> An exploitable use after free vulnerability exists in the window
> function functionality of Sqlite3 3.26.0. A specially crafted SQL
> command can cause a use after free vulnerability, potentially
> resulting in remote code execution. An attacker can send a malicious
> SQL command to trigger this vulnerability.


## Affected packages
 - **sqlite3**
    - **Introduced:** 3.26.0
    - **Fixed:** 3.26.0
