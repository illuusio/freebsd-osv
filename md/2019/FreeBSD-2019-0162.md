# FreeBSD -- Kernel stack disclosure in UFS/FFS
# Problem Description:

A bug causes up to three bytes of kernel stack memory to be written to
disk as uninitialized directory entry padding. This data can be viewed
by any user with read access to the directory. Additionally, a malicious
user with write access to a directory can cause up to 254 bytes of
kernel stack memory to be exposed.

# Impact:

Some amount of the kernel stack is disclosed and written out to the
filesystem.


## Affected packages
 - **FreeBSD-kernel**
    - **Introduced:** 12.0
    - **Fixed:** 12.0_7
    - **Introduced:** 11.2
    - **Fixed:** 11.2_11
