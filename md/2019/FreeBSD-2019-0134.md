# powerdns -- multiple vulnerabilities
PowerDNS Team reports:

> CVE-2019-10162: An issue has been found in PowerDNS Authoritative
> Server allowing an authorized user to cause the server to exit by
> inserting a crafted record in a MASTER type zone under their control.
> The issue is due to the fact that the Authoritative Server will exit
> when it runs into a parsing error while looking up the NS/A/AAAA
> records it is about to use for an outgoing notify.
>
> CVE-2019-10163: An issue has been found in PowerDNS Authoritative
> Server allowing a remote, authorized master server to cause a high CPU
> load or even prevent any further updates to any slave zone by sending
> a large number of NOTIFY messages. Note that only servers configured
> as slaves are affected by this issue.


## Affected packages
 - **powerdns**
    - **Fixed:** 4.1.10
