# mongodb -- Bump Windows package dependencies
Rich Mirch reports:

> An unprivileged user or program on Microsoft Windows which can create
> OpenSSL configuration files in a fixed location may cause utility
> programs shipped with MongoDB server versions less than 4.0.11,
> 3.6.14, and 3.4.22 to run attacker defined code as the user running
> the utility.


## Affected packages
 - **mongodb34**
    - **Fixed:** 3.4.22
 - **mongodb36**
    - **Fixed:** 3.6.14
 - **mongodb40**
    - **Fixed:** 4.0.11
