# uim -- privilege escalation vulnerability
The uim developers reports:

> Takumi ASAKI discovered that uim always trusts environment variables.
> But this is not correct behavior, sometimes environment variables
> shouldn\'t be trusted. This bug causes privilege escalation when
> libuim is linked against setuid/setgid application. Since GTK+
> prohibits setuid/setgid applications, the bug appears only in
> \'immodule for Qt\' enabled Qt. (Normal Qt is also safe.)


## Affected packages
 - **ja-uim**
    - **Fixed:** 0.4.6
