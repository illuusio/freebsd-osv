# up-imapproxy -- multiple vulnerabilities
Timo Sirainen reports:

> There are various bugs in up-imapproxy which can crash it. Since
> up-imapproxy runs in a single process with each connection handled in
> a separate thread, any crash kills all the connections and stops
> listening for new ones.
>
> In 64bit systems it might be possible to make it leak data (mails,
> passwords, ..) from other connections to attacker\'s connection.
> However I don\'t think up-imapproxy actually works in any 64bit system
> so this is just a theoretical problem.


## Affected packages
 - **up-imapproxy**
    - **Fixed:** 1.2.4
 - **pop3proxy**
    - **Fixed:** 1.1
