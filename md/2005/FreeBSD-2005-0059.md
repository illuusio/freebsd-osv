# xpdf -- makeFileKey2() buffer overflow vulnerability
An iDEFENSE Security Advisory reports:

> Remote exploitation of a buffer overflow vulnerability in the xpdf PDF
> viewer included in multiple Unix and Linux distributions could allow
> for arbitrary code execution as the user viewing a PDF file.
>
> The vulnerability specifically exists due to insufficient bounds
> checking while processing a PDF file that provides malicious values in
> the /Encrypt /Length tag. The offending code can be found in the
> `Decrypt::makeFileKey2` function in the source file xpdf/Decrypt.cc.


## Affected packages
 - **xpdf**
    - **Fixed:** 3.00_6
 - **kdegraphics**
    - **Fixed:** 3.3.2_2
 - **gpdf**
    - **Fixed:** 2.8.3
 - **teTeX-base**
    - **Fixed:** 2.0.2_9
 - **cups-base**
    - **Fixed:** 1.1.23.0_3
 - **koffice**
    - **Fixed:** 1.3.5_2,1
 - **pdftohtml**
    - **Fixed:** 0.36_2
