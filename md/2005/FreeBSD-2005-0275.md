# apache -- Certificate Revocation List (CRL) off-by-one vulnerability
Marc Stern reports an off-by-one vulnerability in within mod_ssl. The
vulnerability lies in mod_ssl\'s Certificate Revocation List (CRL). If
Apache is configured to use a CRL this could allow an attacker to crash
a child process causing a Denial of Service.


## Affected packages
 - **apache**
    - **Introduced:** 2.*,1
    - **Fixed:** 2.0.54_1
