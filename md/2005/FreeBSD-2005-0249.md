# gaim -- AIM/ICQ away message buffer overflow
The GAIM team reports:

> A remote AIM or ICQ user can cause a buffer overflow in Gaim by
> setting an away message containing many AIM substitution strings (such
> as %t or %n).


## Affected packages
 - **gaim**
    - **Fixed:** 1.4.0_1
 - **ja-gaim**
    - **Fixed:** 1.4.0_1
 - **ko-gaim**
    - **Fixed:** 1.4.0_1
 - **ru-gaim**
    - **Fixed:** 1.4.0_1
