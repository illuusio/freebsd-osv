# rsnapshot -- local privilege escalation
An rsnapshot Advisory reports:

> The copy_symlink() subroutine in rsnapshot incorrectly changes file
> ownership on the files pointed to by symlinks, not on the symlinks
> themselves. This would allow, under certain circumstances, an
> arbitrary user to take ownership of a file on the main filesystem.


## Affected packages
 - **rsnapshot**
    - **Fixed:** 1.1.7
