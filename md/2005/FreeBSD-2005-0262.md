# pam_ldap -- authentication bypass vulnerability
Luke Howard reports:

> If a pam_ldap client authenticates against an LDAP server that returns
> a passwordPolicyResponse control, but omits the optional \"error\"
> field of the PasswordPolicyResponseValue, then the LDAP authentication
> result will be ignored and the authentication step will always
> succeed.


## Affected packages
 - **pam_ldap**
    - **Fixed:** 1.8.0
