# fetchmail -- denial of service/crash from malicious POP3 server
In fetchmail 6.2.5.1, the remote code injection via POP3 UIDL was fixed,
but a denial of service attack was introduced:

Two possible NULL-pointer dereferences allow a malicious POP3 server to
crash fetchmail by respondig with UID lines containing only the article
number but no UID (in violation of RFC-1939), or a message without
Message-ID when no UIDL support is available.


## Affected packages
 - **fetchmail**
    - **Introduced:** 6.2.5.1
    - **Fixed:** 6.2.5.1
