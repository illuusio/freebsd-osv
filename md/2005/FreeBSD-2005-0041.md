# quake2 -- multiple critical vulnerabilities
An advisory published by Richard Stanway describes numerous critical
vulnerabilities in the Quake II engine:

> Due to unchecked input at various stages in the server, remote users
> are able to cause the server to crash, reveal sensitive information or
> potentially execute arbitrary code.


## Affected packages
 - **quake2forge**
 - **quake2lnx**
    - **Fixed:** 0.16.2
