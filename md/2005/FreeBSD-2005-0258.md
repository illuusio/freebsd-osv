# openvpn -- denial of service: malicious authenticated "tap" client can deplete server virtual memory
James Yonan reports:

> A malicious \[authenticated\] client in \"dev tap\" ethernet bridging
> mode could theoretically flood the server with packets appearing to
> come from hundreds of thousands of different MAC addresses, causing
> the OpenVPN process to deplete system virtual memory as it expands its
> internal routing table.


## Affected packages
 - **openvpn**
    - **Fixed:** 2.0.1
