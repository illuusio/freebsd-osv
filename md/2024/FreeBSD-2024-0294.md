# PostgreSQL -- PL/Perl environment variable changes execute arbitrary code
PostgreSQL project reports:

> Incorrect control of environment variables in PostgreSQL PL/Perl
> allows an unprivileged database user to change sensitive process
> environment variables (e.g. PATH). That often suffices to enable
> arbitrary code execution, even if the attacker lacks a database server
> operating system user.


## Affected packages
 - **postgresql17-plperl**
    - **Fixed:** 17.1
 - **postgresql16-plperl**
    - **Fixed:** 16.5
 - **postgresql15-plperl**
