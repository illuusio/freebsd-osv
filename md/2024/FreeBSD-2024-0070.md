# databases/mongodb* -- Improper Certificate Validation
MongoDB, Inc. reports:

> A security vulnerability was found where a server process running
> MongoDB 3.2.6 or later will allow incoming connections to skip peer
> certificate validation if the server process was started with TLS
> enabled (net.tls.mode set to allowTLS, preferTLS, or requireTLS) and
> without a net.tls.CAFile configured (CVE-2024-1351).


## Affected packages
 - **mongodb44**
    - **Fixed:** 4.4.29
 - **mongodb50**
    - **Fixed:** 5.0.25
 - **mongodb60**
    - **Fixed:** 6.0.14
 - **mongodb70**
    - **Fixed:** 7.0.6
