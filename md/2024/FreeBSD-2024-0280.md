# lrzsz -- Integer overflow in zmodem, crash and information leak
cve@mitre.org reports:

> Lrzsz has an integer overflow vulernability in the src/zm.c:zsdata()
> function. An attacker could exploit this with the sz command to cause
> a crash or potentially leak information to the receiving server.


## Affected packages
 - **lrzsz**
    - **Fixed:** 0.12.20_7
