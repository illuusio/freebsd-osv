# Gitlab -- vulnerabilities
Gitlab reports:

> An attacker can run pipeline jobs as an arbitrary user
>
> Developer user with admin_compliance_framework permission can change
> group URL
>
> Admin push rules custom role allows creation of project level deploy
> token
>
> Package registry vulnerable to manifest confusion
>
> User with admin_group_member permission can ban group members
>
> Subdomain takeover in GitLab Pages


## Affected packages
 - **gitlab-ce**
    - **Introduced:** 17.1.0
    - **Fixed:** 17.1.2
    - **Introduced:** 17.0.0
    - **Fixed:** 17.0.4
    - **Introduced:** 11.8.0
