# Gitlab -- vulnerabilities
Gitlab reports:

> Privilege Escalation via LFS Tokens
>
> DoS through uncontrolled resource consumption when viewing a
> maliciously crafted cargo.toml file
>
> Unintended Access to Usage Data via Scoped Tokens
>
> Gitlab DOS via Harbor registry integration
>
> Resource exhaustion and denial of service with test_report API calls
>
> Streaming endpoint did not invalidate tokens after revocation


## Affected packages
 - **gitlab-ce**
    - **Introduced:** 17.6.0
    - **Fixed:** 17.6.1
    - **Introduced:** 17.5.0
    - **Fixed:** 17.5.3
    - **Introduced:** 8.12.0
    - **Fixed:** 17.4.5
 - **gitlab-ee**
    - **Introduced:** 17.6.0
    - **Fixed:** 17.6.1
    - **Introduced:** 17.5.0
    - **Fixed:** 17.5.3
    - **Introduced:** 8.12.0
    - **Fixed:** 17.4.5
