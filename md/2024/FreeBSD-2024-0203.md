# Configobj -- Regular Expression Denial of Service attack
report@snyk.io reports:

> All versions of the package configobj are vulnerable to Regular
> Expression Denial of Service (ReDoS) via the validate function, using
> (.+?)\\((.\*)\\).\*\*Note:\*\* This is only exploitable in the case of
> a developer putting the offending value in a server side configuration
> file.


## Affected packages
 - **py38-configobj**
    - **Fixed:** 5.0.8
 - **py39-configobj**
    - **Fixed:** 5.0.8
 - **py310-configobj**
    - **Fixed:** 5.0.8
 - **py311-configobj**
    - **Fixed:** 5.0.8
