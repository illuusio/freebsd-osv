# MongoDB -- may be susceptible to DoS due to Accumulated Memory Allocation
cna@mongodb.com reports:

> MongoDB Server may be susceptible to disruption caused by high memory
> usage, potentially leading to server crash. This condition is linked
> to inefficiencies in memory management related to internal operations.
> In scenarios where certain internal processes persist longer than
> anticipated, memory consumption can increase, potentially impacting
> server stability and availability.


## Affected packages
 - **mongodb80**
    - **Fixed:** 8.0.10
