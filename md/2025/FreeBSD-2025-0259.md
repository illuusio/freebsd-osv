# sqlite -- integer overflow
cve-coordination@google.com reports:

> An integer overflow can be triggered in SQLites \`concat_ws()\`
> function. The resulting, truncated integer is then used to allocate a
> buffer. When SQLite then writes the resulting string to the buffer, it
> uses the original, untruncated size and thus a wild Heap Buffer
> overflow of size \~4GB can be triggered. This can result in arbitrary
> code execution.


## Affected packages
 - **sqlite3**
    - **Fixed:** 3.49.1,1
 - **linux-c7-sqlite**
    - **Fixed:** 3.49.1
 - **linux_base-rl9**
    - **Fixed:** 3.49.1
