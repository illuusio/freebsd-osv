# traefik -- Path traversal vulnerability
The traefik project reports:

> There is a potential vulnerability in Traefik managing the requests
> using a PathPrefix, Path or PathRegex matcher. When Traefik is
> configured to route the requests to a backend using a matcher based on
> the path, if the URL contains a URL encoded string in its path, it\'s
> possible to target a backend, exposed using another router, by-passing
> the middlewares chain.


## Affected packages
 - **traefik**
    - **Fixed:** 3.4.1
