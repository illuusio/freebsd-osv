# redis,valkey -- Running Lua function as a different user
redis reports:

> An authenticated user may use a specially crafted Lua script to
> manipulate different LUA objects and potentially run their own code in
> the context of another user The problem exists in all versions of
> Redis with Lua scripting. An additional workaround to mitigate the
> problem without patching the redis-server executable is to prevent
> users from executing Lua scripts. This can be done using ACL to block
> a script by restricting both the EVAL and FUNCTION command families.
