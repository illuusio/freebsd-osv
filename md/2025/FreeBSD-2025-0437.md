# traefik -- Inverted TLS Verification Logic in Kubernetes NGINX Provider
The traefik project reports:

> There is a potential vulnerability in Traefik NGINX provider managing
> the nginx.ingress.kubernetes.io/proxy-ssl-verify annotation. The
> provider inverts the semantics of the
> nginx.ingress.kubernetes.io/proxy-ssl-verify annotation. Setting the
> annotation to \"on\" (intending to enable backend TLS certificate
> verification) actually disables verification, allowing
> man-in-the-middle attacks against HTTPS backends when operators
> believe they are protected.


## Affected packages
 - **traefik**
    - **Fixed:** 3.6.3
