# SQLite < 3.50.3 -- CWE-190 Integer Overflow or Wraparound in FTS5 module
https://github.com/google/security-research/security/advisories/GHSA-v2c8-vqqp-hv3g
reports:

> An integer overflow exists in the FTS5 https://sqlite.org/fts5.html
> extension. It occurs when the size of an array of tombstone pointers
> is calculated and truncated into a 32-bit integer. A pointer to
> partially controlled data can then be written out of bounds.

The FreeBSD build enables the FTS5 extension by default.


## Affected packages
 - **sqlite3**
    - **Introduced:** 3.49.1,1
    - **Fixed:** 3.50.3,1
 - **linux_base-rl9**
 - **linux-c7-sqlite**
    - **Fixed:** 3.50.3
