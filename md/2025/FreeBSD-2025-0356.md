# Erlang - Absolute Path in Zip Module
https://github.com/erlang/otp/security/advisories/GHSA-9g37-pgj9-wrhc
reports:

> Improper Limitation of a Pathname to a Restricted Directory (\'Path
> Traversal\') vulnerability in Erlang OTP (stdlib modules) allows
> Absolute Path Traversal, File Manipulation. This vulnerability is
> associated with program files lib/stdlib/src/zip.erl and program
> routines zip:unzip/1, zip:unzip/2, zip:extract/1, zip:extract/2unless
> the memory option is passed. This issue affects OTP from OTP 17.0
> until OTP28.0.1, OTP27.3.4.1 and OTP26.2.5.13, corresponding to stdlib
> from 2.0 until 7.0.1, 6.2.2.1 and 5.2.3.4.


## Affected packages
 - **erlang**
    - **Introduced:** 17.0
    - **Fixed:** 26.2.5.13,4
 - **erlang-runtime26**
    - **Fixed:** 26.2.5.13
 - **erlang-runtime27**
    - **Fixed:** 27.3.4.1
 - **erlang-runtime28**
    - **Fixed:** 28.0.1
