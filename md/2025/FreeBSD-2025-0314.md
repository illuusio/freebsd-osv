# redis,valkey -- Out of bound read due to a bug in LUA
redis reports:

> An authenticated user may use a specially crafted LUA script to read
> out-of-bound data or crash the server and subsequent denial of
> service. The problem exists in all versions of Redis with Lua
> scripting An additional workaround to mitigate the problem without
> patching the redis-server executable is to prevent users from
> executing Lua scripts. This can be done using ACL to block a script by
> restricting both the EVAL and FUNCTION command families.
