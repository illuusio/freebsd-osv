# redis,valkey -- Lua Use-After-Free may lead to remote code execution
redis reports:

> An authenticated user may use a specially crafted Lua script to
> manipulate the garbage collector, trigger a use-after-free and
> potentially lead to remote code execution. The problem exists in all
> versions of Redis with Lua scripting. An additional workaround to
> mitigate the problem without patching the redis-server executable is
> to prevent users from executing Lua scripts. This can be done using
> ACL to restrict EVAL and EVALSHA commands.
