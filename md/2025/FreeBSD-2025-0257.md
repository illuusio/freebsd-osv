# SQLite -- integer overflow in key info allocation
cve-coordination@google.com reports:

> An integer overflow in the sqlite3KeyInfoFromExprList function in
> SQLite versions 3.39.2 through 3.41.1 allows an attacker with the
> ability to execute arbitrary SQL statements to cause a denial of
> service or disclose sensitive information from process memory via a
> crafted SELECT statement with a large number of expressions in the
> ORDER BY clause.


## Affected packages
 - **sqlite3**
    - **Introduced:** 3.39.2,1
    - **Fixed:** 3.41.2,1
