# Emacs -- Shell injection vulnerability
# Problem Description:

An Emacs user who chooses to invoke elisp-completion-at-point (for code
completion) on untrusted Emacs Lisp source code can trigger unsafe Lisp
macro expansion that allows attackers to execute arbitrary code. This
unsafe expansion also occurs if a user chooses to enable on-the-fly
diagnosis that byte compiles untrusted Emacs Lisp source code.
