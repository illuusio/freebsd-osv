# redis,valkey -- {redis,valkey}-check-aof may lead to stack overflow and potential RCE
Simcha Kosman & CyberArk Labs reports:

> A user can run the {redis,valkeyu}-check-aof cli and pass a long file
> path to trigger a stack buffer overflow, which may potentially lead to
> remote code execution.


## Affected packages
 - **redis**
    - **Introduced:** 8.0.0
    - **Fixed:** 8.0.2
 - **redis74**
    - **Introduced:** 7.4.0
    - **Fixed:** 7.4.4
 - **redis72**
    - **Introduced:** 7.2.0
    - **Fixed:** 7.2.9
 - **valkey**
    - **Fixed:** 8.1.2
