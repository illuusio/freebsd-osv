# redis,valkey -- Lua library commands may lead to integer overflow and potential RCE
redis reports:

> An authenticated user may use a specially crafted Lua script to cause
> an integer overflow and potentially lead to remote code execution The
> problem exists in all versions of Redis with Lua scripting. An
> additional workaround to mitigate the problem without patching the
> redis-server executable is to prevent users from executing Lua
> scripts. This can be done using ACL to block a script by restricting
> both the EVAL and FUNCTION command families.
