# mongodb -- MongoDB Server router will crash when incorrect lsid is set on a sharded query
cna@mongodb.com reports:

> An improper setting of the lsid field on any sharded query can cause a
> crash in MongoDB routers. This issue occurs when a generic argument
> (lsid) is provided in a case when it is not applicable.


## Affected packages
 - **mongodb60**
    - **Fixed:** 6.0.24
 - **mongodb70**
    - **Fixed:** 7.0.18
 - **mongodb80**
    - **Fixed:** 8.0.6
