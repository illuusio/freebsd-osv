# cups -- security vulnerabilities
OpenPrinting reports:

> When the AuthType is set to anything but Basic, if the request
> contains an Authorization: Basic \... header, the password is not
> checked.

> An unsafe deserialization and validation of printer attributes, causes
> null dereference in libcups library.


## Affected packages
 - **cups**
    - **Fixed:** 2.4.13
