# net-snmp -- Remote DoS
The Red Hat Security Response Team reports:

> An array index error, leading to out-of heap-based buffer read flaw
> was found in the way the net-snmp agent performed lookups in the
> extension table. When certain MIB subtrees were handled by the extend
> directive, a remote attacker (having read privileges to the subntree)
> could use this flaw to cause a denial of service condition via an SNMP
> GET request involving a non-existent extension table entry.


## Affected packages
 - **net-snmp**
    - **Fixed:** 5.7.1_7
