# wireshark -- denial of service in DRDA dissector
RedHat security team reports:

> A denial of service flaw was found in the way Distributed Relational
> Database Architecture (DRDA) dissector of Wireshark, a network traffic
> analyzer, performed processing of certain DRDA packet capture files. A
> remote attacker could create a specially-crafted capture file that,
> when opened could lead to wireshark executable to consume excessive
> amount of CPU time and hang with an infinite loop.


## Affected packages
 - **wireshark**
