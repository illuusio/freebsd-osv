# tomcat -- denial of service
The Apache Software Foundation reports:

> When using the NIO connector with sendfile and HTTPS enabled, if a
> client breaks the connection while reading the response an infinite
> loop is entered leading to a denial of service.


## Affected packages
 - **tomcat**
    - **Introduced:** 6.0.0
    - **Fixed:** 6.0.35
 - **tomcat7**
    - **Introduced:** 7.0.0
    - **Fixed:** 7.0.27
