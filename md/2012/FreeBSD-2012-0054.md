# gnutls -- possible overflow/Denial of service vulnerabilities
Mu Dynamics, Inc. reports:

> The block cipher decryption logic in GnuTLS assumed that a record
> containing any data which was a multiple of the block size was valid
> for further decryption processing, leading to a heap corruption
> vulnerability.


## Affected packages
 - **gnutls**
    - **Fixed:** 2.12.18
 - **gnutls-devel**
    - **Introduced:** 2.99,1
    - **Fixed:** 3.0.15
