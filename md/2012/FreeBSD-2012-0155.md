# squidclamav -- cross-site scripting in default virus warning pages
SquidClamav developers report:

> This release fix several security issues by escaping CGI parameters.

Prior to versions 6.7 and 5.8, CGI script clwarn.cgi was not properly
sanitizing input variables, so they could be used to inject arbitrary
strings to the generated page, leading to the cross-site scripting
attacks.


## Affected packages
 - **squidclamav**
    - **Fixed:** 5.8
    - **Introduced:** 6.0
    - **Fixed:** 6.7
