# Tomcat -- Request Smuggling
Apache Tomcat reports:

> If Tomcat was configured to ignore invalid HTTP headers via setting
> rejectIllegalHeader to false (the default for 8.5.x only), Tomcat did
> not reject a request containing an invalid Content-Length header
> making a request smuggling attack possible if Tomcat was located
> behind a reverse proxy that also failed to reject the request with the
> invalid header.
>
> The CVSS score for this vulnerability is 7.5 High
