# go -- multiple vulnerabilities
The Go project reports:

> regexp: stack exhaustion compiling deeply nested expressions
>
> On 64-bit platforms, an extremely deeply nested expression can cause
> regexp.Compile to cause goroutine stack exhaustion, forcing the
> program to exit. Note this applies to very large expressions, on the
> order of 2MB.


## Affected packages
 - **go**
    - **Fixed:** 1.17.8,1
