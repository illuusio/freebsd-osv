# rsync -- client-side arbitrary file write vulnerability
Openwall oss-security reports:

> We have discovered a critical arbitrary file write vulnerability in
> the rsync utility that allows malicious remote servers to write
> arbitrary files inside the directories of connecting peers. The server
> chooses which files/directories are sent to the client. Due to the
> insufficient controls inside the do_server_recv function a malicious
> rysnc server (or Man-in-The-Middle attacker) can overwrite arbitrary
> files in the rsync client target directory and subdirectories.


## Affected packages
 - **rsync**
    - **Fixed:** 3.2.5
