# redis -- Potential remote code execution vulnerability
The Redis core team reports:

> Executing a XAUTOCLAIM command on a stream key in a specific state,
> with a specially crafted COUNT argument, may cause an integer
> overflow, a subsequent heap overflow, and potentially lead to remote
> code execution. The problem affects Redis versions 7.0.0 or newer.


## Affected packages
 - **redis**
    - **Introduced:** 7.0.0
    - **Fixed:** 7.0.5
