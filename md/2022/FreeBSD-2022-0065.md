# gitea -- Improper/incorrect authorization
Youssef Rebahi-Gilbert reports:

> When Gitea is built and configured for PAM authentication it skips
> checking authorization completely. Therefore expired accounts and
> accounts with expired passwords can still login.


## Affected packages
 - **gitea**
    - **Fixed:** 1.16.4
