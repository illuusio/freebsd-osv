# Asterisk -- func_odbc: Possible SQL Injection
The Asterisk project reports:

> Some databases can use backslashes to escape certain characters, such
> as backticks. If input is provided to func_odbc which includes
> backslashes it is possible for func_odbc to construct a broken SQL
> query and the SQL query to fail.


## Affected packages
 - **asterisk16**
    - **Fixed:** 16.25.2
 - **asterisk18**
    - **Fixed:** 18.11.2
