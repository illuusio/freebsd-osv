# varnish -- Request Smuggling Vulnerability
Varnish Cache Project reports:

> A request smuggling attack can be performed on HTTP/1 connections on
> Varnish Cache servers. The smuggled request would be treated as an
> additional request by the Varnish server, go through normal VCL
> processing, and injected as a spurious response on the client
> connection.


## Affected packages
 - **varnish6**
    - **Fixed:** 6.6.2
 - **varnish4**
    - **Fixed:** 4.1.11r6
