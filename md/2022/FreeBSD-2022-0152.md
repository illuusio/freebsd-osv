# FreeBSD -- AIO credential reference count leak
# Problem Description:

The aio_aqueue function, used by the lio_listio system call, fails to
release a reference to a credential in an error case.

# Impact:

An attacker may cause the reference count to overflow, leading to a use
after free (UAF).


## Affected packages
 - **FreeBSD-kernel**
    - **Introduced:** 13.0
    - **Fixed:** 13.0_12
    - **Introduced:** 12.3
    - **Fixed:** 12.3_6
