# lighttpd -- Remote DOS in CRLF parsing
Lighttpd SA:

> If the connection aborts during parsing \"\\r\\n\\r\\n\" the server
> might get into a infinite loop and use 100% of the CPU time. lighttpd
> still responses to other requests. This can be repeated until either
> the server limit for concurrent connections or file descriptors is
> reached.
>
> The bug was reported and fixed by Robert Jakabosky.


## Affected packages
 - **lighttpd**
    - **Introduced:** 1.4.11,1
    - **Fixed:** 1.4.13_2
