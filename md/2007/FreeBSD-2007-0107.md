# id3lib -- insecure temporary file creation
Debian Bug report log reports:

> When tagging file \$foo, a temporary copy of the file is created, and
> for some reason, libid3 doesn\'t use mkstemp but just creates
> \$foo.XXXXXX literally, without any checking.
>
> This would silently truncate and overwrite an existing \$foo.XXXXXX.


## Affected packages
 - **id3lib**
    - **Fixed:** 3.8.3_4
