# c-ares -- DNS Cache Poisoning Vulnerability
Secunia reports:

> The vulnerability is caused due to predictable DNS \"Transaction ID\"
> field in DNS queries and can be exploited to poison the DNS cache of
> an application using the library if a valid ID is guessed.


## Affected packages
 - **c-ares**
    - **Fixed:** 1.4.0
