# w3m -- format string vulnerability
An anonymous person reports:

> w3m-0.5.1 crashes when using the -dump or -backend options to open a
> HTTPS URL with a SSL certificate where the CN contains
> \"%n%n%n%n%n%n\".


## Affected packages
 - **w3m**
    - **Fixed:** 0.5.1_6
 - **w3m-img**
    - **Fixed:** 0.5.1_6
 - **w3m-m17n**
    - **Fixed:** 0.5.1_6
 - **w3m-m17n-img**
    - **Fixed:** 0.5.1_6
 - **ja-w3m**
    - **Fixed:** 0.5.1_6
 - **ja-w3m-img**
    - **Fixed:** 0.5.1_6
