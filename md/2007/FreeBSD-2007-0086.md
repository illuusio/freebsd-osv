# rsync -- off by one stack overflow
BugTraq reports:

> The rsync utility is prone to an off-by-one buffer-overflow
> vulnerability. This issue is due to a failure of the application to
> properly bounds-check user-supplied input.
>
> Successfully exploiting this issue may allow arbitrary code-execution
> in the context of the affected utility.


## Affected packages
 - **rsync**
    - **Fixed:** 2.6.9_1
