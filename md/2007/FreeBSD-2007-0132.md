# phpmyadmin -- cross-site scripting vulnerability
The DigiTrust Group reports:

> When creating a new database, a malicious user can use a client-side
> Web proxy to place malicious code in the db parameter of the POST
> request. Since db_create.php does not properly sanitize user-supplied
> input, an administrator could face a persistent XSS attack when the
> database names are displayed.


## Affected packages
 - **phpMyAdmin**
    - **Fixed:** 2.11.2.1
