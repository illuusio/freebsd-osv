# mod_perl -- remote DoS in PATH_INFO parsing
Mandriva reports:

> PerlRun.pm in Apache mod_perl 1.29 and earlier, and RegistryCooker.pm
> in mod_perl 2.x, does not properly escape PATH_INFO before use in a
> regular expression, which allows remote attackers to cause a denial of
> service (resource consumption) via a crafted URI.


## Affected packages
 - **mod_perl**
    - **Fixed:** 1.30
 - **mod_perl2**
    - **Fixed:** 2.0.3_2,3
