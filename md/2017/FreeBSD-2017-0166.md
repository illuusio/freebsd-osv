# proftpd -- user chroot escape vulnerability
NVD reports:

> ProFTPD \... controls whether the home directory of a user could
> contain a symbolic link through the AllowChrootSymlinks configuration
> option, but checks only the last path component when enforcing
> AllowChrootSymlinks. Attackers with local access could bypass the
> AllowChrootSymlinks control by replacing a path component (other than
> the last one) with a symbolic link.


## Affected packages
 - **proftpd**
    - **Fixed:** 1.3.5e
