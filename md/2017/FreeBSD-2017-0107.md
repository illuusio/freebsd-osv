# libetpan -- null dereference vulnerability in MIME parsing component
rwhitworth reports:

> I was using American Fuzzy Lop (afl-fuzz) to fuzz input to the
> mime-parse test program. Is fixing these crashes something you\'re
> interested in? The input files can be found here:
> https://github.com/rwhitworth/libetpan-fuzz/. The files can be
> executed as ./mime-parse id_filename to cause seg faults.
