# xrdp -- local user can cause a denial of service
xrdp reports:

> The scp_v0s_accept function in the session manager uses an untrusted
> integer as a write length, which allows local users to cause a denial
> of service (buffer overflow and application crash) or possibly have
> unspecified other impact via a crafted input stream.


## Affected packages
 - **xrdp-devel**
    - **Fixed:** 0.9.3,1
    - **Introduced:** 0.9.3_1,1,1
    - **Fixed:** 0.9.4,1
