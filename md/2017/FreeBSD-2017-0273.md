# asterisk -- Memory/File Descriptor/RTP leak in pjsip session resource
The Asterisk project reports:

> A memory leak occurs when an Asterisk pjsip session object is created
> and that call gets rejected before the session itself is fully
> established. When this happens the session object never gets
> destroyed. This then leads to file descriptors and RTP ports being
> leaked as well.


## Affected packages
 - **asterisk13**
    - **Introduced:** 13.5.0
    - **Fixed:** 13.18.1
