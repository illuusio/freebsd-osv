# exim -- remote code execution, deny of service in BDAT
Exim team reports:

> The receive_msg function in receive.c in the SMTP daemon in Exim 4.88
> and 4.89 allows remote attackers to execute arbitrary code or cause a
> denial of service (use-after-free) via vectors involving BDAT
> commands.


## Affected packages
 - **exim**
    - **Introduced:** 4.88
    - **Fixed:** 4.89_2
