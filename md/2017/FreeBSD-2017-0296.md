# asterisk -- DOS Vulnerability in Asterisk chan_skinny
The Asterisk project reports:

> If the chan_skinny (AKA SCCP protocol) channel driver is flooded with
> certain requests it can cause the asterisk process to use excessive
> amounts of virtual memory eventually causing asterisk to stop
> processing requests of any kind.


## Affected packages
 - **asterisk13**
    - **Fixed:** 13.18.3
