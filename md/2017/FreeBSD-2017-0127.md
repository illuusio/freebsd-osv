# FreeRADIUS -- TLS resumption authentication bypass
Stefan Winter reports:

> The TLS session cache in FreeRADIUS before 3.0.14 fails to reliably
> prevent resumption of an unauthenticated session, which allows remote
> attackers (such as malicious 802.1X supplicants) to bypass
> authentication via PEAP or TTLS.


## Affected packages
 - **freeradius**
    - **Fixed:** 3.0.14
 - **freeradius2**
    - **Fixed:** 3.0.14
 - **freeradius3**
    - **Fixed:** 3.0.14
