# osip -- Improper Restriction of Operations within the Bounds of a Memory Buffer
osip developers reports:

> In libosip2 in GNU oSIP 4.1.0 and 5.0.0, a malformed SIP message can
> lead to a heap buffer overflow in the msg_osip_body_parse() function
> defined in osipparser2/osip_message_parse.c, resulting in a remote
> DoS.


## Affected packages
 - **libosip2**
    - **Fixed:** 5.0.0
