# ez-ipupdate -- format string vulnerability
Data supplied by a remote server is used as the format string instead of
as parameters in a syslog() call. This may lead to crashes or potential
running of arbitrary code. It is only a problem when running in daemon
mode (very common) and when using some service types.


## Affected packages
 - **ez-ipupdate**
    - **Fixed:** 3.0.11b8_2
