# leafnode fetchnews denial-of-service triggered by truncated transmission
When a downloaded news article ends prematurely, i. e. when the server
sends \[CR\]LF.\[CR\]LF before sending a blank line, fetchnews may wait
indefinitely for data that never arrives. Workaround: configure
\"minlines=1\" (or use a bigger value) in the configuration file. Found
by Toni Viemer√∂.


## Affected packages
 - **leafnode**
    - **Fixed:** 1.9.47
