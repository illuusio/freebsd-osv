# gaim -- Content-Length header denial-of-service vulnerability
Sean "infamous42md" reports:

> When a remote server provides a large \"content-length\" header value,
> Gaim will attempt to allocate a buffer to store the content, however
> this allocation attempt will cause Gaim to crash if the length exceeds
> the amount of possible memory. This happens when reading profile
> information on some protocols. It also happens when smiley themes are
> installed via drag and drop.
