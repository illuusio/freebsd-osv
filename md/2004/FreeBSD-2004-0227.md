# gd -- integer overflow
infamous41md reports about the GD Graphics Library:

> There is an integer overflow when allocating memory in the routine
> that handles loading PNG image files. This later leads to heap data
> structures being overwritten. If an attacker tricked a user into
> loading a malicious PNG image, they could leverage this into executing
> arbitrary code in the context of the user opening image.
