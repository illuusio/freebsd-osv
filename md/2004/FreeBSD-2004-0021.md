# libxml2 stack buffer overflow in URI parsing
Yuuichi Teranishi reported a crash in libxml2\'s URI handling when a
long URL is supplied. The implementation in nanohttp.c and nanoftp.c
uses a 4K stack buffer, and longer URLs will overwrite the stack. This
could result in denial-of-service or arbitrary code execution in
applications using libxml2 to parse documents.


## Affected packages
 - **libxml2**
    - **Fixed:** 2.6.6
