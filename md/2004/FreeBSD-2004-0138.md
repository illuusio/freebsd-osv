# SpamAssassin -- denial-of-service in tokenize_headers
According to the SpamAssassin 2.64 release announcement:

> Security fix prevents a denial of service attack open to certain
> malformed messages; this DoS affects all SpamAssassin 2.5x and 2.6x
> versions to date.

The issue appears to be triggered by overly long message headers.


## Affected packages
 - **p5-Mail-SpamAssassin**
    - **Fixed:** 2.64
