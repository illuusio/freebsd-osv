# Pavuk HTTP Location header overflow
When pavuk sends a request to a web server and the server sends back the
HTTP status code 305 (Use Proxy), pavuk copies data from the HTTP
Location header in an unsafe manner. This leads to a stack-based buffer
overflow with control over EIP.


## Affected packages
 - **pavuk**
    - **Fixed:** 0.9.28_5
