# a2ps -- insecure command line argument handling
Rudolf Polzer reports:

> a2ps builds a command line for file() containing an unescaped version
> of the file name, thus might call external programs described by the
> file name. Running a cronjob over a public writable directory a2ps-ing
> all files in it - or simply typing \"a2ps \*.txt\" in /tmp - is
> therefore dangerous.


## Affected packages
 - **a2ps-a4**
    - **Fixed:** 4.13b_2
 - **a2ps-letter**
    - **Fixed:** 4.13b_2
 - **a2ps-letterdj**
    - **Fixed:** 4.13b_2
