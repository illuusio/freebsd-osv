# rssh & scponly -- arbitrary command execution
Jason Wies identified both rssh & scponly have a vulnerability that
allows arbitrary command execution. He reports:

> The problem is compounded when you recognize that the main use of rssh
> and scponly is to allow file transfers, which in turn allows a
> malicious user to transfer and execute entire custom scripts on the
> remote machine.


## Affected packages
 - **rssh**
    - **Fixed:** 2.2.2
 - **scponly**
    - **Fixed:** 4.0
