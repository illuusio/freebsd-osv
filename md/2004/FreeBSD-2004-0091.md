# neon date parsing vulnerability
Stefan Esser reports:

> A vulnerability within a libneon date parsing function could cause a
> heap overflow which could lead to remote code execution, depending on
> the application using libneon.

The vulnerability is in the function ne_rfc1036_parse, which is in turn
used by the function ne_httpdate_parse. Applications using either of
these neon functions may be vulnerable.


## Affected packages
 - **neon**
    - **Fixed:** 0.24.5_1
 - **sitecopy**
    - **Fixed:** 0.13.4_1
