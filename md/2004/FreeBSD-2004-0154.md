# apache -- apr_uri_parse IPv6 address handling vulnerability
The Apache Software Foundation Security Team discovered a programming
error in the apr-util library function apr_uri_parse. When parsing IPv6
literal addresses, it is possible that a length is incorrectly
calculated to be negative, and this value is passed to memcpy. This may
result in an exploitable vulnerability on some platforms, including
FreeBSD.


## Affected packages
 - **apache**
    - **Introduced:** 2.0
    - **Fixed:** 2.0.50_3
