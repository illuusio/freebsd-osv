# Apache 1.3 IP address access control failure on some 64-bit platforms
Henning Brauer discovered a programming error in Apache 1.3\'s
mod_access that results in the netmasks in IP address access control
rules being interpreted incorrectly on 64-bit, big-endian platforms. In
some cases, this could cause a \`deny from\' IP address access control
rule including a netmask to fail.


## Affected packages
 - **apache**
    - **Fixed:** 1.3.29_2
 - **apache+mod_ssl**
    - **Fixed:** 1.3.29+2.8.16_1
 - **apache+ssl**
    - **Fixed:** 1.3.29.1.53_1
 - **ru-apache**
    - **Fixed:** 1.3.29+30.19_1
 - **ru-apache+mod_ssl**
    - **Fixed:** 1.3.29+30.19+2.8.16_1
