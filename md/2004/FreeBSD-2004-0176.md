# mozilla -- hostname spoofing bug
When processing URIs that contain an unqualified host name\--
specifically, a domain name of only one component\-- Mozilla will
perform matching against the first component of the domain name in SSL
certificates. In other words, in some situations, a certificate issued
to \"www.example.com\" will be accepted as matching \"www\".
