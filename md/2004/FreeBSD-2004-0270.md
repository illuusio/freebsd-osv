# mysql -- erroneous access restrictions applied to table renames
A Red Hat advisory reports:

> Oleksandr Byelkin discovered that \"ALTER TABLE \... RENAME\" checked
> the CREATE/INSERT rights of the old table instead of the new one.

Table access restrictions, on the affected MySQL servers, may accidently
or intentially be bypassed due to this bug.


## Affected packages
 - **mysql-server**
    - **Fixed:** 3.23.58_3
    - **Introduced:** 4.*
    - **Fixed:** 4.0.21
