# xpdf -- buffer overflow vulnerability
An iDEFENSE Security Advisory reports:

> Remote exploitation of a buffer overflow vulnerability in the xpdf PDF
> viewer, as included in multiple Linux distributions, could allow
> attackers to execute arbitrary code as the user viewing a PDF file.
> The offending code can be found in the Gfx::doImage() function in the
> source file xpdf/Gfx.cc.


## Affected packages
 - **xpdf**
    - **Fixed:** 3.00_5
 - **kdegraphics**
    - **Fixed:** 3.3.2_1
 - **gpdf**
    - **Fixed:** 2.8.1
 - **teTeX-base**
    - **Fixed:** 2.0.2_6
 - **cups-base**
    - **Fixed:** 1.1.22.0
 - **koffice**
    - **Fixed:** 1.3.5,1
 - **pdftohtml**
    - **Fixed:** 0.36_1
