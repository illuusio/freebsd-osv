# FreeBSD -- Insufficient oce(4) ioctl(2) privilege checking
# Problem Description:

The driver-specific ioctl(2) command handlers in oce(4) failed to check
whether the caller has sufficient privileges to perform the
corresponding operation.

# Impact:

The oce(4) handler permits unprivileged users to send passthrough
commands to device firmware.


## Affected packages
 - **FreeBSD-kernel**
    - **Introduced:** 12.1
    - **Fixed:** 12.1_3
    - **Introduced:** 11.3
    - **Fixed:** 11.3_7
