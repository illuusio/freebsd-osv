# powerdns -- Leaking uninitialised memory through crafted zone records
PowerDNS Team reports

> CVE-2020-17482: An issue has been found in PowerDNS Authoritative
> Server before 4.3.1 where an authorized user with the ability to
> insert crafted records into a zone might be able to leak the content
> of uninitialized memory. Such a user could be a customer inserting
> data via a control panel, or somebody with access to the REST API.
> Crafted records cannot be inserted via AXFR.


## Affected packages
 - **powerdns**
    - **Introduced:** 4.3.0
    - **Fixed:** 4.3.1
    - **Introduced:** 4.2.0
    - **Fixed:** 4.2.3
    - **Introduced:** 4.1.0
