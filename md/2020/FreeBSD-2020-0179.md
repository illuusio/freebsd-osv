# FreeBSD -- IPv6 socket option race condition and use after free
# Problem Description:

The IPV6_2292PKTOPTIONS set handler was missing synchronization, so
racing accesses could modify freed memory.

# Impact:

A malicious user application could trigger memory corruption, leading to
privilege escalation.


## Affected packages
 - **FreeBSD-kernel**
    - **Introduced:** 12.1
    - **Fixed:** 12.1_7
    - **Introduced:** 11.4
    - **Fixed:** 11.4_1
    - **Introduced:** 11.3
    - **Fixed:** 11.3_11
