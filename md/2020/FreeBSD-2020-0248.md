# FreeBSD -- bhyve SVM guest escape
# Problem Description:

A number of AMD virtualization instructions operate on host physical
addresses, are not subject to nested page table translation, and guest
use of these instructions was not trapped.

# Impact:

From kernel mode a malicious guest can write to arbitrary host memory
(with some constraints), affording the guest full control of the host.


## Affected packages
 - **FreeBSD-kernel**
    - **Introduced:** 12.1
    - **Fixed:** 12.1_10
    - **Introduced:** 11.4

    - **Introduced:** 11.3
