# ansible - subversion password leak from PID
Borja Tarraso reports:

> A flaw was found in Ansible 2.7.16 and prior, 2.8.8 and prior, and
> 2.9.5 and prior when a password is set with the argument \"password\"
> of svn module, it is used on svn command line, disclosing to other
> users within the same node. An attacker could take advantage by
> reading the cmdline file from that particular PID on the procfs.


## Affected packages
