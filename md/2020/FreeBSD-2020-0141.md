# FreeBSD -- USB HID descriptor parsing error
# Problem Description:

If the push/pop level of the USB HID state is not restored within the
processing of the same HID item, an invalid memory location may be used
for subsequent HID item processing.

# Impact:

An attacker with physical access to a USB port may be able to use a
specially crafted USB device to gain kernel or user-space code
execution.


## Affected packages
 - **FreeBSD-kernel**
    - **Introduced:** 12.1
    - **Fixed:** 12.1_6
    - **Introduced:** 11.3
    - **Fixed:** 11.3_10
