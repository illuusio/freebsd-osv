# glpi -- Multiple SQL Injections Stemming From isNameQuoted()
MITRE Corporation reports:

> In GLPI before version 9.5.2, when supplying a back tick in input that
> gets put into a SQL query,the application does not escape or sanitize
> allowing for SQL Injection to occur. Leveraging this vulnerability an
> attacker is able to exfiltrate sensitive information like passwords,
> reset tokens, personal details, and more. The issue is patched in
> version 9.5.2


## Affected packages
 - **glpi**
    - **Fixed:** 9.5.2,1
