# FreeBSD -- SCTP socket use-after-free bug
# Problem Description:

Due to improper handling in the kernel, a use-after-free bug can be
triggered by sending large user messages from multiple threads on the
same socket.

# Impact:

Triggering the use-after-free situation may result in unintended kernel
behaviour including a kernel panic.

 - **FreeBSD-kernel**
