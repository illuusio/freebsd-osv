# glpi -- weak csrf tokens
MITRE Corporation reports:

> In GLPI after version 0.83.3 and before version 9.4.6, the CSRF tokens
> are generated using an insecure algorithm. The implementation uses
> rand and uniqid and MD5 which does not provide secure values. This is
> fixed in version 9.4.6.


## Affected packages
 - **glpi**
    - **Introduced:** 0.83.3,1
    - **Fixed:** 9.4.6,1
