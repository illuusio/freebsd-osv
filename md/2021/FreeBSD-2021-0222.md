# go -- archive/zip: overflow in preallocation check can cause OOM panic
The Go project reports:

> An oversight in the previous fix still allows for an OOM panic when
> the indicated directory size in the archive header is so large that
> subtracting it from the archive size overflows a uint64, effectively
> bypassing the check that the number of files in the archive is
> reasonable.


## Affected packages
 - **go**
    - **Fixed:** 1.17.1,1
