# squashfs-tools -- Integer overflow
Phillip Lougher reports:

> Integer overflow in the read_fragment_table_4 function in unsquash-4.c
> in Squashfs and sasquatch allows remote attackers to cause a denial of
> service (application crash) via a crafted input, which triggers a
> stack-based buffer overflow.


## Affected packages
 - **squashfs-tools**
    - **Fixed:** 4.4
