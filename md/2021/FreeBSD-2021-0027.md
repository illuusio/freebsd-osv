# FreeBSD -- Xen guests can triger backend Out Of Memory
# Problem Description:

Some OSes (including Linux, FreeBSD, and NetBSD) are processing watch
events using a single thread. If the events are received faster than the
thread is able to handle, they will get queued.

As the queue is unbound, a guest may be able to trigger a OOM in the
backend.
