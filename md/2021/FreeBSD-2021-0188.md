# redis -- Integer overflow issues with BITFIELD command on 32-bit systems
Huang Zhw reports:

> On 32-bit versions, Redis BITFIELD command is vulnerable to integer
> overflow that can potentially be exploited to corrupt the heap, leak
> arbitrary heap contents or trigger remote code execution. The
> vulnerability involves constructing specially crafted bit commands
> which overflow the bit offset.
>
> This problem only affects 32-bit versions of Redis.


## Affected packages
 - **redis**
    - **Fixed:** 6.0.15
 - **redis-devel**
    - **Fixed:** 6.2.5
 - **redis5**
    - **Fixed:** 5.0.13
