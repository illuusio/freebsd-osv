# FreeBSD -- login.access fails to apply rules
# Problem Description:

A regression in the login.access(5) rule processor has the effect of
causing rules to fail to match even when they should not. This means
that rules denying access may be ignored.

# Impact:

The configuration in login.access(5) may not be applied, permitting
login access to users even when the system is configured to deny it.


## Affected packages
 - **FreeBSD**
    - **Introduced:** 12.2
    - **Fixed:** 12.2_4
    - **Introduced:** 11.4
    - **Fixed:** 11.4_8
