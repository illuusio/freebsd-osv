# redis -- Integer overflow on 32-bit systems
Redis Development team reports:

> Redis 4.0 or newer uses a configurable limit for the maximum supported
> bulk input size. By default, it is 512MB which is a safe value for all
> platforms. If the limit is significantly increased, receiving a large
> request from a client may trigger several integer overflow scenarios,
> which would result with buffer overflow and heap corruption.


## Affected packages
 - **redis-devel**
    - **Fixed:** 6.2.0
 - **redis**
    - **Fixed:** 6.0.11
 - **redis5**
    - **Fixed:** 5.0.11
