# aiohttp -- open redirect vulnerability
Sviatoslav Sydorenko reports:

> Open redirect vulnerability --- a maliciously crafted link to an
> aiohttp-based web-server could redirect the browser to a different
> website.
>
> It is caused by a bug in the
> `aiohttp.web_middlewares.normalize_path_middleware` middleware.


## Affected packages
 - **py36-aiohttp**
    - **Fixed:** 3.7.3
 - **py37-aiohttp**
    - **Fixed:** 3.7.3
 - **py38-aiohttp**
    - **Fixed:** 3.7.3
 - **py39-aiohttp**
    - **Fixed:** 3.7.3
