# SOGo -- SAML user authentication impersonation
sogo.nu reports:

> SOGo was not validating the signatures of any SAML assertions it
> received.
>
> This means any actor with network access to the deployment could
> impersonate
>
> users when SAML was the authentication method.


## Affected packages
 - **sogo**
    - **Fixed:** 5.1.1
 - **sogo-activesync**
    - **Fixed:** 5.1.1
 - **sogo2**
    - **Fixed:** 2.4.1
 - **sogo2-activesync**
    - **Fixed:** 2.4.1
