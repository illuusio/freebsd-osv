# fetchmail -- 6.4.19 and older denial of service or information disclosure
Matthias Andree reports:

> When a log message exceeds c. 2 kByte in size, for instance, with very
> long header contents, and depending on verbosity option, fetchmail can
> crash or misreport each first log message that requires a buffer
> reallocation.


## Affected packages
 - **fetchmail**
    - **Fixed:** 6.3.9
    - **Introduced:** 6.3.17
    - **Fixed:** 6.4.20
