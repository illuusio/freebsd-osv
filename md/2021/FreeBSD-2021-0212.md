# fetchmail -- STARTTLS bypass vulnerabilities
Problem:

> In certain circumstances, fetchmail 6.4.21 and older would not encrypt
> the session using STARTTLS/STLS, and might not have cleared session
> state across the TLS negotiation.


## Affected packages
 - **fetchmail**
    - **Fixed:** 6.4.22.r1
