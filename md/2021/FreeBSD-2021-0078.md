# ruby -- XML round-trip vulnerability in REXML
Juho Nurminen reports:

> When parsing and serializing a crafted XML document, REXML gem
> (including the one bundled with Ruby) can create a wrong XML document
> whose structure is different from the original one. The impact of this
> issue highly depends on context, but it may lead to a vulnerability in
> some programs that are using REXML.


## Affected packages
 - **ruby**
    - **Introduced:** 2.5.0,1
    - **Fixed:** 2.5.9,1
    - **Introduced:** 2.6.0,1
    - **Fixed:** 2.6.7,1
    - **Introduced:** 2.7.0,1
    - **Fixed:** 2.7.3,1
    - **Introduced:** 3.0.0.p1,1
    - **Fixed:** 3.0.1,1
 - **rubygem-rexml**
    - **Fixed:** 3.2.5
