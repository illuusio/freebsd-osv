# lasso -- signature checking failure
entrouvert reports:

> When AuthnResponse messages are not signed (which is permitted by the
> specifiation), all assertion\'s signatures should be checked, but
> currently after the first signed assertion is checked all following
> assertions are accepted without checking their signature, and the last
> one is considered the main assertion.


## Affected packages
 - **lasso**
    - **Fixed:** 2.7.0
