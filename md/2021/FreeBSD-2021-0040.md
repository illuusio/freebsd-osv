# asterisk -- Remote crash in res_pjsip_diversion
The Asterisk project reports:

> If a registered user is tricked into dialing a malicious number that
> sends lots of 181 responses to Asterisk, each one will cause a 181 to
> be sent back to the original caller with an increasing number of
> entries in the \"Supported\" header. Eventually the number of entries
> in the header exceeds the size of the entry array and causes a crash.
