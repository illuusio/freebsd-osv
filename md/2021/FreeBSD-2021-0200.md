# PostgreSQL server -- Memory disclosure in certain queries
The PostgreSQL Project reports:

> A purpose-crafted query can read arbitrary bytes of server memory. In
> the default configuration, any authenticated database user can
> complete this attack at will. The attack does not require the ability
> to create objects. If server settings include max_worker_processes=0,
> the known versions of this attack are infeasible. However,
> undiscovered variants of the attack may be independent of that
> setting.


## Affected packages
 - **postgresql13-server**
    - **Fixed:** 13.4
 - **postgresql12-server**
    - **Fixed:** 12.8
 - **postgresql11-server**
    - **Fixed:** 11.13
