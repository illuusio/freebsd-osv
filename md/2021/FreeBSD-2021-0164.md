# dovecot-pigeonhole -- Sieve excessive resource usage
Dovecot team reports reports:

> Sieve interpreter is not protected against abusive scripts that claim
> excessive resource usage. Fixed by limiting the user CPU time per
> single script execution and cumulatively over several script runs
> within a configurable timeout period. Sufficiently large CPU time
> usage is summed in the Sieve script binary and execution is blocked
> when the sum exceeds the limit within that time. The block is lifted
> when the script is updated after the resource usage times out.


## Affected packages
 - **dovecot-pigeonhole**
    - **Fixed:** 0.5.15
