# openslp -- denial of service vulnerability
Qinghao Tang reports:

> The function ParseExtension() in openslp 1.2.1 contains vulnerability:
> an attacker can cause a denial of service (infinite loop) via a packet
> with crafted \"nextoffset\" value and \"extid\" value.


## Affected packages
 - **openslp**
    - **Fixed:** 2.0.0
