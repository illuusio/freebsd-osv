# cpio -- multiple vulnerabilities
From the Debian Security Team:

> Heap-based buffer overflow in the process_copy_in function in GNU Cpio
> 2.11 allows remote attackers to cause a denial of service via a large
> block value in a cpio archive.

> cpio 2.11, when using the \--no-absolute-filenames option, allows
> local users to write to arbitrary files via a symlink attack on a file
> in an archive.


## Affected packages
 - **gcpio**
    - **Fixed:** 2.11_3
