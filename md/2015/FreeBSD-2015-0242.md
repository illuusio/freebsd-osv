# py-foolscap -- local file inclusion
Brian Warner reports:

> The \"flappserver\" feature was found to have a vulnerability in the
> service-lookup code which, when combined with an attacker who has the
> ability to write files to a location where the flappserver process
> could read them, would allow that attacker to obtain control of the
> flappserver process.


## Affected packages
 - **py27-foolscap**
    - **Fixed:** 0.7.0
 - **py32-foolscap**
    - **Fixed:** 0.7.0
 - **py33-foolscap**
    - **Fixed:** 0.7.0
 - **py34-foolscap**
    - **Fixed:** 0.7.0
