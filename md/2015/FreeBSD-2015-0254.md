# freexl -- integer overflow
Stefan Cornelius reports:

> There\'s an integer overflow in the allocate_cells() function when
> trying to allocate the memory for worksheet with specially crafted
> row/column dimensions. This can be exploited to cause a heap memory
> corruption. The most likely outcome of this is a crash when trying to
> initialize the cells later in the function.


## Affected packages
 - **freexl**
    - **Fixed:** 1.0.2
