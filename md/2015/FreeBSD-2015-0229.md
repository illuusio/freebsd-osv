# net-snmp -- snmptrapd crash
Murray McAllister reports:

> A remote denial-of-service flaw was found in the way snmptrapd handled
> certain SNMP traps when started with the \"-OQ\" option. If an
> attacker sent an SNMP trap containing a variable with a NULL type
> where an integer variable type was expected, it would cause snmptrapd
> to crash.
