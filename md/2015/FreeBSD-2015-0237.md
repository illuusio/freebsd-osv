# lighttpd -- Log injection vulnerability in mod_auth
MITRE reports:

> mod_auth in lighttpd before 1.4.36 allows remote attackers to inject
> arbitrary log entries via a basic HTTP authentication string without a
> colon character, as demonstrated by a string containing a NULL and new
> line character.


## Affected packages
 - **lighttpd**
    - **Fixed:** 1.4.36
