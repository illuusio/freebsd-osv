# PHPmailer -- SMTP injection vulnerability
PHPMailer changelog reports:

> Fix vulnerability that allowed email addresses with line breaks (valid
> in RFC5322) to pass to SMTP, permitting message injection at the SMTP
> level. Mitigated in both the address validator and in the lower-level
> SMTP class. Thanks to Takeshi Terada.


## Affected packages
 - **phpmailer**
    - **Fixed:** 5.2.14
