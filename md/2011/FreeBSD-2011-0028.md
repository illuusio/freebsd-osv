# openoffice.org -- Multiple vulnerabilities
OpenOffice.org Security Team reports:

> Fixed in OpenOffice.org 3.3
>
> - [CVE-2010-2935 /
>   CVE-2010-2936](http://www.openoffice.org/security/cves/CVE-2010-2935_CVE-2010-2936.html):
>   Security Vulnerability in OpenOffice.org related to PowerPoint
>   document processing
> - [CVE-2010-3450](http://www.openoffice.org/security/cves/CVE-2010-3450.html):
>   Security Vulnerability in OpenOffice.org related to Extensions and
>   filter package files
> - [CVE-2010-3451 /
>   CVE-2010-3452](http://www.openoffice.org/security/cves/CVE-2010-3451_CVE-2010-3452.html):
>   Security Vulnerability in OpenOffice.org related to RTF document
>   processing
> - [CVE-2010-3453 /
>   CVE-2010-3454](http://www.openoffice.org/security/cves/CVE-2010-3453_CVE-2010-3454.html):
>   Security Vulnerability in OpenOffice.org related to Word document
>   processing
> - [CVE-2010-3689](http://www.openoffice.org/security/cves/CVE-2010-3689.html):
>   Insecure LD_LIBRARY_PATH usage in OpenOffice.org shell scripts
> - [CVE-2010-3702 /
>   CVE-2010-3704](http://www.openoffice.org/security/cves/CVE-2010-3702_CVE-2010-3704.html):
>   Security Vulnerability in OpenOffice.org\'s PDF Import extension
>   resulting from 3rd party library XPDF
> - [CVE-2010-4008 /
>   CVE-2010-4494](http://www.openoffice.org/security/cves/CVE-2010-4008_CVE-2010-4494.html):
>   Possible Security Vulnerability in OpenOffice.org resulting from 3rd
>   party library LIBXML2
> - [CVE-2010-4253](http://www.openoffice.org/security/cves/CVE-2010-4253.html):
>   Security Vulnerability in OpenOffice.org related to PNG file
>   processing
> - [CVE-2010-4643](http://www.openoffice.org/security/cves/CVE-2010-4643.html):
>   Security Vulnerability in OpenOffice.org related to TGA file
>   processing


## Affected packages
 - **openoffice.org**
    - **Fixed:** 3.3.0
