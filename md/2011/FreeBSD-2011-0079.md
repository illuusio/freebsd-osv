# Erlang -- ssh library uses a weak random number generator
US-CERT reports:

> The Erlang/OTP ssh library implements a number of cryptographic
> operations that depend on cryptographically strong random numbers.
> Unfortunately the RNG used by the library is not cryptographically
> strong, and is further weakened by the use of predictable seed
> material. The RNG (Wichman-Hill) is not mixed with an entropy source.


## Affected packages
 - **erlang**
    - **Fixed:** r14b03
