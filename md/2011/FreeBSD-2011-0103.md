# libXfont -- possible local privilege escalation
Tomas Hoger reports:

> The compress/ LZW decompress implentation does not correctly handle
> compressed streams that contain code words that were not yet added to
> the decompression table. This may lead to arbitrary memory corruption.
> Successfull exploitation may possible lead to a local privilege
> escalation.


## Affected packages
 - **libXfont**
    - **Fixed:** 1.4.4_1,1
