# OpenTTD -- Denial of service via improperly validated commands
The OpenTTD Team reports:

> Multiple off-by-one errors in order_cmd.cpp in OpenTTD before 1.1.3
> allow remote attackers to cause a denial of service (daemon crash) or
> possibly execute arbitrary code via a crafted CMD_INSERT_ORDER
> command.


## Affected packages
 - **openttd**
    - **Introduced:** 0.3.5
    - **Fixed:** 1.1.3
