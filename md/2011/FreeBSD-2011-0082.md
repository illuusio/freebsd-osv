# asterisk -- Remote crash vulnerability
The Asterisk Development Team reports:

> If a remote user initiates a SIP call and the recipient picks up, the
> remote user can reply with a malformed Contact header that Asterisk
> will improperly handle and cause a crash due to a segmentation fault.


## Affected packages
 - **asterisk18**
    - **Introduced:** 1.8.*,1
    - **Fixed:** 1.8.4.2
