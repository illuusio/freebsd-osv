# postgresql81-server -- SET ROLE privilege escalation
The PostgreSQL team reports:

> Due to inadequate validity checking, a user could exploit the special
> case that SET ROLE normally uses to restore the previous role setting
> after an error. This allowed ordinary users to acquire superuser
> status, for example.


## Affected packages
 - **postgresql-server**
    - **Introduced:** 8.1.0
    - **Fixed:** 8.1.3
