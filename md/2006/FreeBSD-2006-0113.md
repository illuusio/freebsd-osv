# libxine -- buffer overflow vulnerability
A Secunia Advisory reports:

> Federico L. Bossi Bonin has discovered a weakness in xine-lib, which
> can be exploited by malicious people to crash certain applications on
> a user\'s system.
>
> The weakness is cause due to a heap corruption within the
> \"xineplug_inp_http.so\" plugin when handling an overly large reply
> from the HTTP server. This can be exploited to crash an application
> that uses the plugin (e.g. gxine).


## Affected packages
 - **libxine**
    - **Fixed:** 1.1.1_6
