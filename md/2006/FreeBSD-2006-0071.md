# mplayer -- Multiple integer overflows
Secunia reports:

> The vulnerabilities are caused due to integer overflow errors in
> \"libmpdemux/asfheader.c\" within the handling of an ASF file, and in
> \"libmpdemux/aviheader.c\" when parsing the \"indx\" chunk in an AVI
> file. This can be exploited to cause heap-based buffer overflows via a
> malicious ASF file, or via a AVI file with specially-crafted
> \"wLongsPerEntry\" and \"nEntriesInUse\" values in the \"indx\" chunk.


## Affected packages
 - **mplayer**
    - **Fixed:** 0.99.7_12
 - **mplayer-esound**
    - **Fixed:** 0.99.7_12

    - **Fixed:** 0.99.7_12
