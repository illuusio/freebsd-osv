# openldap -- slapd acl selfwrite Security Issue
Howard Chu reports:

> An ACL of the form \'access to dn.subtree=\"ou=groups,
> dc=example,dc=com\" attr=member by \* selfwrite\' is intended to only
> allow users to add/delete their own DN to the target attribute.
> Currently it allows any DNs to be modified.


## Affected packages
 - **openldap-server**
    - **Fixed:** 2.3.25
 - **openldap-sasl-server**
    - **Fixed:** 2.3.25
