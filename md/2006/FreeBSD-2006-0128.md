# samba -- memory exhaustion DoS in smbd
The Samba Team reports:

> The smbd daemon maintains internal data structures used track active
> connections to file and printer shares. In certain circumstances an
> attacker may be able to continually increase the memory usage of an
> smbd process by issuing a large number of share connection requests.
> This defect affects all Samba configurations.


## Affected packages
 - **samba**
    - **Introduced:** 3.0.1,1
    - **Fixed:** 3.0.23,1
 - **ja-samba**
    - **Introduced:** 3.0.1,1
    - **Fixed:** 3.0.23,1
