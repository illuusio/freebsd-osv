# libmusicbrainz -- multiple buffer overflow vulnerabilities
SecurityFocus reports about libmusicbrainz:

> The libmusicbrainz library is prone to multiple buffer-overflow
> vulnerabilities because the application fails to check the size of the
> data before copying it into a finite-sized internal memory buffer.
>
> An attacker can exploit these issues to execute arbitrary code within
> the context of the application or to cause a denial-of-service
> condition.


## Affected packages
 - **libmusicbrainz**
    - **Fixed:** 2.1.3
