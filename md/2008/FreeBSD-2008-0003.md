# maradns -- CNAME record resource rotation denial of service
Secunia reports:

> A vulnerability has been reported in MaraDNS, which can be exploited
> by malicious people to cause a Denial of Service.
>
> The vulnerability is caused due to an error within the handling of
> certain DNS packets. This can be exploited to cause a resource
> rotation by sending specially crafted DNS packets, which cause an
> authoritative CNAME record to not resolve, resulting in a Denial of
> Sevices.


## Affected packages
 - **maradns**
    - **Fixed:** 1.2.12.08
