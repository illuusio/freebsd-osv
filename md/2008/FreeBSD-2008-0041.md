# libxine -- array index vulnerability
xine Team reports:

> A new xine-lib version is now available. This release contains a
> security fix (an unchecked array index that could allows remote
> attackers to execute arbitrary code via a header structure containing
> a negative offset, which is used to dereference a function pointer.)


## Affected packages
 - **libxine**
    - **Fixed:** 1.1.12
