# rubygem-rails -- SQL injection vulnerability
Jonathan Weiss reports, that it is possible to perform an SQL injection
in Rails applications via not correctly sanitized :limit and :offset
parameters. It is possible to change arbitrary values in affected tables
or gain access to the sensitive data.


## Affected packages
 - **rubygem-rails**
    - **Fixed:** 2.2.2
