# mplayer -- twinvq processing buffer overflow vulnerability
A trapkit reports:

> MPlayer contains a stack buffer overflow vulnerability while parsing
> malformed TwinVQ media files. The vulnerability may be exploited by a
> (remote) attacker to execute arbitrary code in the context of MPlayer.


## Affected packages
 - **mplayer**
    - **Fixed:** 0.99.11_9
 - **mplayer-esound**
    - **Fixed:** 0.99.11_9
 - **mplayer-gtk**
    - **Fixed:** 0.99.11_9
 - **mplayer-gtk-esound**
    - **Fixed:** 0.99.11_9
 - **mplayer-gtk2**
    - **Fixed:** 0.99.11_9
 - **mplayer-gtk2-esound**
    - **Fixed:** 0.99.11_9
