# streamripper -- multiple buffer overflows
Secunia reports:

> A boundary error exists within http_parse_sc_header() in lib/http.c
> when parsing an overly long HTTP header starting with \"Zwitterion
> v\".
>
> A boundary error exists within http_get_pls() in lib/http.c when
> parsing a specially crafted pls playlist containing an overly long
> entry.
>
> A boundary error exists within http_get_m3u() in lib/http.c when
> parsing a specially crafted m3u playlist containing an overly long
> \"File\" entry.


## Affected packages
 - **streamripper**
    - **Fixed:** 1.64.0
