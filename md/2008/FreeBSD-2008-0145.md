# imlib2 -- XPM processing buffer overflow vulnerability
Secunia reports:

> A vulnerability has been discovered in imlib2, which can be exploited
> by malicious people to potentially compromise an application using the
> library.
>
> The vulnerability is caused due to a pointer arithmetic error within
> the \"load()\" function provided by the XPM loader. This can be
> exploited to cause a heap-based buffer overflow via a specially
> crafted XPM file.
>
> Successful exploitation may allow execution of arbitrary code.


## Affected packages
 - **imlib2**
    - **Fixed:** 1.4.1.000_1,2
 - **imlib2-nox11**
    - **Fixed:** 1.4.1.000_1,2
