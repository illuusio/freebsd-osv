# gtar -- GNU TAR safer_name_suffix Remote Denial of Service Vulnerability
SecurityFocus reports:

> GNUs tar and cpio utilities are prone to a denial-of-service
> vulnerability because of insecure use of the alloca() function.
>
> Successfully exploiting this issue allows attackers to crash the
> affected utilities and possibly to execute code but this has not been
> confirmed.


## Affected packages
 - **gtar**
    - **Fixed:** 1.19
