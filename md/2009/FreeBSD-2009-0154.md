# elinks -- buffer overflow vulnerability
SecurityFocus reports:

> ELinks is prone to an off-by-one buffer-overflow vulnerability because
> the application fails to accurately reference the last element of a
> buffer.
>
> Attackers may leverage this issue to execute arbitrary code in the
> context of the application. Failed attacks will cause
> denial-of-service conditions.


## Affected packages
 - **elinks**
    - **Fixed:** 0.11.4
