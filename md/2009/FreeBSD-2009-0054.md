# phppgadmin -- directory traversal with register_globals enabled
Secunia reports:

> Dun has discovered a vulnerability in phpPgAdmin, which can be
> exploited by malicious people to disclose sensitive information.
>
> Input passed via the \"\_language\" parameter to libraries/lib.inc.php
> is not properly sanitised before being used to include files. This can
> be exploited to include arbitrary files from local resources via
> directory traversal attacks and URL-encoded NULL bytes.


## Affected packages
 - **phppgadmin**
    - **Fixed:** 4.2.2
