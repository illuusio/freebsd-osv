# mysql -- empty bit-string literal denial of service
MySQL reports:

> The vulnerability is caused due to an error when processing an empty
> bit-string literal and can be exploited to crash the server via a
> specially crafted SQL statement.


## Affected packages
 - **mysql-server**
    - **Introduced:** 5.0
    - **Fixed:** 5.0.66
    - **Introduced:** 5.1
    - **Fixed:** 5.1.26
    - **Introduced:** 6.0
    - **Fixed:** 6.0.6
