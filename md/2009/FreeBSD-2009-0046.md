# pycrypto -- ARC2 module buffer overflow
Dwayne C. Litzenberger reports:

> pycrypto is exposed to a buffer overflow issue because it fails to
> adequately verify user-supplied input. This issue resides in the ARC2
> module. This issue can be triggered with specially crafted ARC2 keys
> in excess of 128 bytes.


## Affected packages
 - **py-pycrypto**
    - **Fixed:** 2.0.1_2
