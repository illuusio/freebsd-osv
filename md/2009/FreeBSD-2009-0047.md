# dia -- remote command execution vulnerability
Security Focus reports:

> An attacker could exploit this issue by enticing an unsuspecting
> victim to execute the vulnerable application in a directory containing
> a malicious Python file. A successful exploit will allow arbitrary
> Python commands to run within the privileges of the currently
> logged-in user.


## Affected packages
 - **dia**
    - **Fixed:** 0.96.1_6,1
