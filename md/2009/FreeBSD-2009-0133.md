# dnsmasq -- TFTP server remote code injection vulnerability
Simon Kelley reports:

> Fix security problem which allowed any host permitted to do TFTP to
> possibly compromise dnsmasq by remote buffer overflow when TFTP
> enabled.
>
> Fix a problem which allowed a malicious TFTP client to crash dnsmasq.


## Affected packages
 - **dnsmasq**
    - **Fixed:** 2.50
