# Gitlab -- multiple vulnerabilities
GitLab reports:

> # SSRF in services and web hooks
>
> There were multiple server-side request forgery issues in the Services
> feature. An attacker could make requests to servers within the same
> network of the GitLab instance. This could lead to information
> disclosure, authentication bypass, or potentially code execution. This
> issue has been assigned
> [CVE-2018-8801](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2018-8801).
>
> # Gitlab Auth0 integration issue
>
> There was an issue with the GitLab `omniauth-auth0` configuration
> which resulted in the Auth0 integration signing in the wrong users.


## Affected packages
 - **gitlab**
    - **Introduced:** 10.5.0
    - **Fixed:** 10.5.6
    - **Introduced:** 10.4.0
    - **Fixed:** 10.4.6
    - **Introduced:** 8.3
    - **Fixed:** 10.3.9
