# mailman -- hardening against malicious listowners injecting evil HTML scripts
Mark Sapiro reports:

> Existing protections against malicious listowners injecting evil
> scripts into listinfo pages have had a few more checks added.
>
> A few more error messages have had their values HTML escaped.
>
> The hash generated when SUBSCRIBE_FORM_SECRET is set could have been
> the same as one generated at the same time for a different list and IP
> address.


## Affected packages
 - **mailman**
    - **Fixed:** 2.1.27
 - **mailman-with-htdig**
    - **Fixed:** 2.1.27
 - **ja-mailman**
    - **Fixed:** 2.1.14.j7_5,1
