# asterisk and pjsip -- multiple vulnerabilities
The Asterisk project reports:

> AST-2018-002 - By crafting an SDP message with an invalid media format
> description Asterisk crashes when using the pjsip channel driver
> because pjproject\'s sdp parsing algorithm fails to catch the invalid
> media format description.
>
> AST-2018-003 - By crafting an SDP message body with an invalid fmtp
> attribute Asterisk crashes when using the pjsip channel driver because
> pjproject\'s fmtp retrieval function fails to check if fmtp value is
> empty (set empty if previously parsed as invalid).


## Affected packages
 - **asterisk13**
    - **Fixed:** 13.19.2
 - **pjsip**
    - **Fixed:** 2.7.2
 - **pjsip-extsrtp**
    - **Fixed:** 2.7.2
