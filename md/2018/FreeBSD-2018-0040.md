# bchunk -- heap-based buffer overflow (with invalid free) and crash
Mitre reports:

> bchunk 1.2.0 and 1.2.1 is vulnerable to a heap-based buffer overflow
> (with a resultant invalid free) and crash when processing a malformed
> CUE (.cue) file.


## Affected packages
 - **bchunk**
    - **Introduced:** 1.2.0
    - **Fixed:** 1.2.1
