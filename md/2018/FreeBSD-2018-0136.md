# Git -- Fix memory out-of-bounds and remote code execution vulnerabilities (CVE-2018-11233 and CVE-2018-11235)
The Git community reports:

> - In affected versions of Git, code to sanity-check pathnames on NTFS
>   can result in reading out-of-bounds memory.
> - In affected versions of Git, remote code execution can occur. With a
>   crafted .gitmodules file, a malicious project can execute an
>   arbitrary script on a machine that runs \"git clone
>   \--recurse-submodules\" because submodule \"names\" are obtained
>   from this file, and then appended to \$GIT_DIR/modules, leading to
>   directory traversal with \"../\" in a name. Finally, post-checkout
>   hooks from a submodule are executed, bypassing the intended design
>   in which hooks are not obtained from a remote server.
