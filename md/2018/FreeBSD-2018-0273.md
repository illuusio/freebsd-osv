# asterisk -- Remote crash vulnerability DNS SRV and NAPTR lookups
The Asterisk project reports:

> There is a buffer overflow vulnerability in dns_srv and dns_naptr
> functions of Asterisk that allows an attacker to crash Asterisk via a
> specially crafted DNS SRV or NAPTR response. The attackers request
> causes Asterisk to segfault and crash.


## Affected packages
 - **asterisk15**
    - **Fixed:** 15.6.1
 - **asterisk16**
    - **Fixed:** 16.0.1
