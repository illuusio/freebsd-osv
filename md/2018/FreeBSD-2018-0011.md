# unbound -- vulnerability in the processing of wildcard synthesized NSEC records
Unbound reports:

> We discovered a vulnerability in the processing of wildcard
> synthesized NSEC records. While synthesis of NSEC records is allowed
> by RFC4592, these synthesized owner names should not be used in the
> NSEC processing. This does, however, happen in Unbound 1.6.7 and
> earlier versions.


## Affected packages
 - **unbound**
    - **Fixed:** 1.6.8
