# Memory leak in different components
MITRE reports:

> bsixel 1.8.1 has a memory leak in sixel_decoder_decode in decoder.c,
> image_buffer_resize in fromsixel.c, sixel_decode_raw in fromsixel.c
> and sixel_allocator_new in allocator.c


## Affected packages
 - **libsixel**
    - **Fixed:** 1.8.2,1
