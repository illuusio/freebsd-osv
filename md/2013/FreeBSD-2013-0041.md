# rubygem-dragonfly -- arbitrary code execution
Mark Evans reports:

> Unfortnately there is a security vulnerability in Dragonfly when used
> with Rails which would potentially allow an attacker to run arbitrary
> code on a host machine using carefully crafted requests.


## Affected packages
 - **rubygem18-dragonfly**
    - **Fixed:** 0.9.14
 - **rubygem19-dragonfly**
    - **Fixed:** 0.9.14
 - **rubygem20-dragonfly**
    - **Fixed:** 0.9.14
