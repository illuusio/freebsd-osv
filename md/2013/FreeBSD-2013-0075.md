# phpMyAdmin -- XSS due to unescaped HTML output in GIS visualisation page
The phpMyAdmin development team reports:

> When modifying a URL parameter with a crafted value it is possible to
> trigger an XSS.
>
> These XSS can only be triggered when a valid database is known and
> when a valid cookie token is used.


## Affected packages
 - **phpMyAdmin**
    - **Introduced:** 3.5
    - **Fixed:** 3.5.8
