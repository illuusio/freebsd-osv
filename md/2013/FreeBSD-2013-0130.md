# subversion -- remotely triggerable "Assertion failed" DoS vulnerability or read overflow.
Subversion Project reports:

> Subversion\'s mod_dav_svn Apache HTTPD server module will trigger an
> assertion on some requests made against a revision root. This can lead
> to a DoS. If assertions are disabled it will trigger a read overflow
> which may cause a SEGFAULT (or equivalent) or undefined behavior.
>
> Commit access is required to exploit this.


## Affected packages
 - **subversion**
    - **Introduced:** 1.8.0
    - **Fixed:** 1.8.1
    - **Introduced:** 1.7.0
    - **Fixed:** 1.7.11
