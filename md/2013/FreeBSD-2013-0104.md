# phpMyAdmin -- XSS due to unescaped HTML output in Create View page
The phpMyAdmin development team reports:

> When creating a view with a crafted name and an incorrect CREATE
> statement, it is possible to trigger an XSS.
>
> This vulnerability can be triggered only by someone who logged in to
> phpMyAdmin, as the usual token protection prevents non-logged-in users
> from accessing the required form.


## Affected packages
 - **phpMyAdmin**
    - **Introduced:** 4.0
    - **Fixed:** 4.0.3
