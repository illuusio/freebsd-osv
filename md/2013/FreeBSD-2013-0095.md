# devel/subversion -- fsfs repositories can be corrupted by newline characters in filenames
Subversion team reports:

> If a filename which contains a newline character (ASCII 0x0a) is
> committed to a repository using the FSFS format, the resulting
> revision is corrupt.


## Affected packages
 - **subversion**
    - **Introduced:** 1.7.0
    - **Fixed:** 1.7.10
    - **Introduced:** 1.1.0
    - **Fixed:** 1.6.23
