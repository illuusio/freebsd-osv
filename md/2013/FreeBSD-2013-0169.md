# Quassel IRC -- SQL injection vulnerability
Quassel IRC developers report:

> SQL injection vulnerability in Quassel IRC before 0.9.1, when Qt 4.8.5
> or later and PostgreSQL 8.2 or later are used, allows remote attackers
> to execute arbitrary SQL commands via a \\ (backslash) in a message.


## Affected packages
 - **quassel**
    - **Fixed:** 0.9.1
