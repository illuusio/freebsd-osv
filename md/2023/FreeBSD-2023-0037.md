# TightVNC -- Muliple Vulnerabilities
MITRE reports:

> TightVNC code version 1.3.10 contains global buffer overflow in
> HandleCoRREBBP macro function, which can potentially result code
> execution. This attack appear to be exploitable via network
> connectivity.

> TightVNC code version 1.3.10 contains global buffer overflow in
> HandleCoRREBBP macro function, which can potentially result code
> execution. This attack appear to be exploitable via network
> connectivity.

> TightVNC code version 1.3.10 contains heap buffer overflow in
> InitialiseRFBConnection function, which can potentially result code
> execution. This attack appear to be exploitable via network
> connectivity.

> TightVNC code version 1.3.10 contains null pointer dereference in
> HandleZlibBPP function, which results Denial of System (DoS). This
> attack appear to be exploitable via network connectivity.


## Affected packages
 - **tightvnc**
    - **Fixed:** 1.3.10_6
