# redis -- Possible bypassing Unix socket permissions
Redis core team reports:

> The wrong order of listen(2) and chmod(2) calls creates a race
> condition that can be used by another process to bypass desired Unix
> socket permissions on startup.


## Affected packages
 - **redis**
    - **Fixed:** 7.2.2
 - **redis-devel**
    - **Fixed:** 7.2.2.20231018
 - **redis70**
    - **Fixed:** 7.0.14
 - **redis62**
    - **Fixed:** 6.2.14
