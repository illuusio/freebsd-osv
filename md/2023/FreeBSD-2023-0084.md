# rack -- possible denial of service vulnerability in header parsing
ooooooo_q reports:

> Carefully crafted input can cause header parsing in Rack to take an
> unexpected amount of time, possibly resulting in a denial of service
> attack vector. Any applications that parse headers using Rack
> (virtually all Rails applications) are impacted.


## Affected packages
 - **rubygem-rack**
    - **Fixed:** 3.0.6.1,3
 - **rubygem-rack22**
    - **Fixed:** 2.2.6.6,3
 - **rubygem-rack16**
    - **Fixed:** 1.6.14
