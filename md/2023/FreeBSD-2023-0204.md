# redis -- heap overflow in COMMAND GETKEYS and ACL evaluation
Redis core team reports:

> Extracting key names from a command and a list of arguments may, in
> some cases, trigger a heap overflow and result in reading random heap
> memory, heap corruption and potentially remote code execution.
> Specifically: using COMMAND GETKEYS\* and validation of key names in
> ACL rules.


## Affected packages
 - **redis**
    - **Fixed:** 7.0.12
 - **redis-devel**
    - **Fixed:** 7.0.12.20230710
