# py39-joblib -- arbitrary code execution
jimlinntu reports:

> The package joblib from 0 and before 1.2.0 are vulnerable to Arbitrary
> Code Execution via the pre_dispatch flag in Parallel() class due to
> the eval() statement.


## Affected packages
 - **py39-joblib**
    - **Fixed:** 1.2.0
