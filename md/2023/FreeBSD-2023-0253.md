# py-flask-caching -- remote code execution or local privilege escalation vulnerabilities
subnix reports:

> The Flask-Caching extension through 2.0.2 for Flask relies on Pickle
> for serialization, which may lead to remote code execution or local
> privilege escalation.
>
> If an attacker gains access to cache storage (e.g., filesystem,
> Memcached, Redis, etc.), they can construct a crafted payload, poison
> the cache, and execute Python code.
