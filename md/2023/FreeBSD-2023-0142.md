# mod_gnutls -- Infinite Loop on request read timeout
The mod_gnutls project reports:

> Mod_gnutls is a TLS module for Apache HTTPD based on GnuTLS. Versions
> from 0.9.0 to 0.12.0 (including) did not properly fail blocking read
> operations on TLS connections when the transport hit timeouts. Instead
> it entered an endless loop retrying the read operation, consuming CPU
> resources. This could be exploited for denial of service attacks. If
> trace level logging was enabled, it would also produce an excessive
> amount of log output during the loop, consuming disk space.


## Affected packages
 - **ap24-mod_gnutls**
    - **Fixed:** 0.12.1
