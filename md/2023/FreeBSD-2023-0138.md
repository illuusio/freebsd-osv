# py-beaker -- arbitrary code execution vulnerability
matheusbrat reports:

> The Beaker library through 1.12.1 for Python is affected by
> deserialization of untrusted data, which could lead to arbitrary code
> execution.


## Affected packages
 - **py37-beaker**
    - **Fixed:** 1.12.1
 - **py38-beaker**
    - **Fixed:** 1.12.1
 - **py39-beaker**
    - **Fixed:** 1.12.1
 - **py310-beaker**
    - **Fixed:** 1.12.1
 - **py311-beaker**
    - **Fixed:** 1.12.1
