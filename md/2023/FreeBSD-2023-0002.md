# rxvt-unicode is vulnerable to a remote code execution
Marc Lehmann reports:

> The biggest issue is resolving CVE-2022-4170, which allows command
> execution inside urxvt from within the terminal (that means anything
> that can output text in the terminal can start commands in the context
> of the urxvt process, even remotely).


## Affected packages
 - **rxvt-unicode**
    - **Fixed:** 9.31
