# freerdp -- clients using the `/video` command line switch might read uninitialized data
MITRE reports:

> All FreeRDP based clients when using the \`/video\` command line
> switch might read uninitialized data, decode it as audio/video and
> display the result. FreeRDP based server implementations are not
> affected.


## Affected packages
 - **freerdp**
    - **Fixed:** 2.8.1
