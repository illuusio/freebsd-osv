# py-Flask-Cors -- directory traversal vulnerability
praetorian-colby-morgan reports:

> An issue was discovered in Flask-CORS (aka CORS Middleware for Flask)
> before 3.0.9.
>
> It allows ../ directory traversal to access private resources because
> resource matching does not ensure that pathnames are in a canonical
> format.


## Affected packages
 - **py37-Flask-Cors**
    - **Fixed:** 3.0.9
 - **py38-Flask-Cors**
    - **Fixed:** 3.0.9
 - **py39-Flask-Cors**
    - **Fixed:** 3.0.9
 - **py310-Flask-Cors**
    - **Fixed:** 3.0.9
 - **py311-Flask-Cors**
    - **Fixed:** 3.0.9
