# varnish -- HTTP/2 Rapid Reset Attack
Varnish Cache Project reports:

> A denial of service attack can be performed on Varnish Cache servers
> that have the HTTP/2 protocol turned on. An attacker can create a
> large volume of streams and immediately reset them without ever
> reaching the maximum number of concurrent streams allowed for the
> session, causing the Varnish server to consume unnecessary resources
> processing requests for which the response will not be delivered.


## Affected packages
 - **varnish7**
    - **Fixed:** 7.4.2
 - **varnish6**
    - **Fixed:** 6.6.3
