# py-markdown2 -- XSS vulnerability
TheGrandPew reports:

> python-markdown2 through 2.3.8 allows XSS because element names are
> mishandled unless a \\w+ match succeeds.
>
> For example, an attack might use elementname@ or elementname- with an
> onclick attribute.


## Affected packages
 - **py37-markdown2**
    - **Fixed:** 2.3.9
 - **py38-markdown2**
    - **Fixed:** 2.3.9
 - **py39-markdown2**
    - **Fixed:** 2.3.9
 - **py310-markdown2**
    - **Fixed:** 2.3.9
 - **py311-markdown2**
    - **Fixed:** 2.3.9
