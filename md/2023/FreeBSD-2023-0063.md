# redis -- multiple vulnerabilities
The Redis core team reports:

> 
>
> CVE-2023-25155
> :   Specially crafted SRANDMEMBER, ZRANDMEMBER, and HRANDFIELD
>     commands can trigger an integer overflow, resulting in a runtime
>     assertion and termination of the Redis server process.
>
> CVE-2022-36021
> :   String matching commands (like SCAN or KEYS) with a specially
>     crafted pattern to trigger a denial-of-service attack on Redis,
>     causing it to hang and consume 100% CPU time.


## Affected packages
 - **redis**
    - **Fixed:** 7.0.9
 - **redis-devel**
    - **Fixed:** 7.0.9.20230228
 - **redis62**
    - **Fixed:** 6.2.11
 - **redis6**
    - **Fixed:** 6.0.18
