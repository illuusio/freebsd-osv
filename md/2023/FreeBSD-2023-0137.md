# py-psutil -- double free vulnerability
ret2libc reports:

> psutil (aka python-psutil) through 5.6.5 can have a double free.
>
> This occurs because of refcount mishandling within a while or for loop
> that converts system data into a Python object.


## Affected packages
 - **py37-psutil121**
    - **Fixed:** 5.6.6
 - **py38-psutil121**
    - **Fixed:** 5.6.6
 - **py39-psutil121**
    - **Fixed:** 5.6.6
 - **py310-psutil121**
    - **Fixed:** 5.6.6
 - **py311-psutil121**
    - **Fixed:** 5.6.6
