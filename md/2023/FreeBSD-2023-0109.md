# py39-configobj -- vulnerable to Regular Expression Denial of Service
DarkTinia reports:

> All versions of the package configobj are vulnerable to Regular
> Expression Denial of Service (ReDoS) via the validate function, using
> (.+?)\\((.\*)\\).
>
> \*\*Note:\*\* This is only exploitable in the case of a developer,
> putting the offending value in a server side configuration file.


## Affected packages
 - **py39-configobj**
    - **Fixed:** 5.0.6_1
