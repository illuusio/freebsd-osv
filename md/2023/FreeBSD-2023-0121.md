# py-cinder -- data leak
Duncan Thomas reports:

> The (1) GlusterFS and (2) Linux Smbfs drivers in OpenStack Cinder
> before 2014.1.3 allows remote authenticated users to obtain file data
> from the Cinder-volume host by cloning and attaching a volume with a
> crafted qcow2 header.
