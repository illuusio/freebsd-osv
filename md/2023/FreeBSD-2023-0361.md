# Gitlab -- vulnerabilities
Gitlab reports:

> Smartcard authentication allows impersonation of arbitrary user using
> user\'s public certificate
>
> When subgroup is allowed to merge or push to protected branches,
> subgroup members with the Developer role may gain the ability to push
> or merge
>
> The GitLab web interface does not ensure the integrity of information
> when downloading the source code from installation packages or tags
>
> Project maintainer can escalate to Project owner using project access
> token rotate API
>
> Omission of Double Encoding in File Names Facilitates the Creation of
> Repositories with Malicious Content
>
> Unvalidated timeSpent value leads to unable to load issues on Issue
> board
>
> Developer can bypass predefined variables via REST API
>
> Auditor users can create merge requests on projects they don\'t have
> access to


## Affected packages
 - **gitlab-ce**
    - **Introduced:** 16.6.0
    - **Fixed:** 16.6.2
    - **Introduced:** 16.5.0
    - **Fixed:** 16.5.4
    - **Introduced:** 8.17.0
    - **Fixed:** 16.4.4
